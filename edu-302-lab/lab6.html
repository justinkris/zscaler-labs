Lab 6: Configure ZPA Logging & Reporting

LAB 6

Scenario: Configure the ZPA Log Streaming Service to stream User Activity and App Connector Status logs to the on-premises SIEM over TCP 514, using the Corp: Log Streaming Connector.

Time: ~30 min
Portal: ZPA Admin Portal
💭 THINK FIRST · ZPA LOG STREAMING

LSS is ZPA's equivalent of NSS — it bridges the cloud's access logs to your on-premises SIEM. The key difference: LSS uses an App Connector (not a separate VM) as the forwarding agent.

  • Why does ZPA use an App Connector as the log streaming agent rather than a separate dedicated VM like ZIA's NSS?
    Q1 — MODEL ANSWER
    ZPA's App Connector already has a secure, persistent outbound tunnel to the Zscaler cloud. Reusing this tunnel for log streaming avoids the need for a separate VM with its own cloud connectivity and certificate management. The "Corp: Log Streaming Connector" is simply an App Connector whose role is dedicated to forwarding logs — it can be the same connector used for app access, or a separate instance. This reduces infrastructure complexity and cost.
  • What are the two types of ZPA logs that Safemarch is streaming via LSS, and what different security questions does each answer?
    Q2 — MODEL ANSWER
    (1) User Activity logs (Client Connector users only, CSV format) — answer "who accessed which private app, when, from which device, with what result?" These are the ZPA equivalent of ZIA web logs. (2) App Connector Status logs (CSV format) — answer "is my App Connector healthy, and if an app became unreachable, when did the connector lose connectivity?" Status logs are operational/availability logs, not access logs. Together they give a full picture: user-level access accountability + infrastructure health.
  • What happens if the TCP port 514 between the Log Streaming Connector (10.0.0.20) and the SIEM Server (10.0.0.2) is blocked by a firewall?
    Q3 — MODEL ANSWER
    LSS logs are never delivered — the App Connector establishes the ZPA cloud connection and receives logs, but cannot forward them to the SIEM. The connector's log queue may fill and drop records. Unlike NSS (which has a dedicated service interface), LSS relies on the App Connector's own network interface to reach the SIEM, so TCP 514 must be permitted between the connector's IP (10.0.0.20) and the SIEM (10.0.0.2). This is a common oversight when firewall rules are written only for the NSS VM IPs.

Configuration Requirements

Work through all three requirements in the ZPA Admin Portal. Check each one off as you complete it.

LSS Configuration · ZPA Admin Portal → Administration → Log Streaming Service
  1. Log in to the ZPA Admin Portal
  2. Navigate to Logs → Log Streaming → Log Receivers
  3. Click Add
  4. Log Receiver tab — enter a name (e.g. LSS-UserActivity), set Domain or IP Address to 10.0.0.20, TCP Port to 514, leave TLS Encryption as Disabled, select Corp: Log Streaming Connector under App Connector Groups → click Next
  5. Log Stream tab — set Log Type to User Activity, Log Template to CSV, review the Log Stream Content fields (default is fine for most deployments) → click Next
  6. Review tab — confirm all settings, then click Save

Each log receiver supports one log stream type — create a separate receiver for App Connector Status.

  1. Navigate to Logs → Log Streaming → Log Receivers
  2. Click Add to create a second log receiver
  3. Log Receiver tab — enter a name (e.g. LSS-ConnectorStatus), set Domain or IP Address to 10.0.0.20, TCP Port to 514, TLS Encryption Disabled, select Corp: Log Streaming Connector under App Connector Groups → click Next
  4. Log Stream tab — set Log Type to App Connector Status, Log Template to CSV, review the Log Stream Content → click Next
  5. Review tab — confirm all settings, then click Save
  1. From the Corp: Client PC VM, open a browser and connect to a private application segment configured in Lab 5
  2. Return to the ZPA Admin Portal and navigate to Logs → Log Streaming → Log Receivers — confirm both LSS-UserActivity and LSS-ConnectorStatus show a Connected status
  3. On the SIEM VM at 10.0.0.20, check the incoming log stream (e.g. tail -f /var/log/syslog or your SIEM's event viewer) and confirm User Activity records are arriving with fields: user, application, bytes transferred, and policy rule
  4. Confirm App Connector Status records are also arriving — look for entries with tunnel state and connector health fields
  5. Verify the log format matches the CSV template configured in each receiver

Testing Requirements

  1. Perform ZPA activity: connect to an application segment from the Corp: Client PC VM.
  2. On the SIEM VM at 10.0.0.20, confirm User Activity log records are arriving.
  3. Confirm App Connector Status records are also arriving.
  4. Verify log format matches expected fields.

MENTAL NOTE
LSS has three components: (1) the Log Receiver (the SIEM destination — 10.0.0.20 TCP 514), (2) User Activity logs (per-session detail — who reached what and when), (3) App Connector Status logs (connector health — essential for detecting outages before users report them). Both log types use the same receiver but are separate streaming policies.
