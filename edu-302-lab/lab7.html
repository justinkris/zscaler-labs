Lab 7: Protect the Leaking of HIPAA Documents

LAB 7

Scenario: Configure a ZIA DLP policy to inspect all file upload traffic and block uploads containing HIPAA-sensitive medical information. Test the policy using a Medical Info PDF from the DLP Test Files folder.

Time: ~30 min
Portal: ZIA Admin (https://admin.zscalerthree.net)
💭 THINK FIRST · DATA LOSS PREVENTION

DLP works by pattern matching content against dictionaries of known sensitive data signatures — but it can only inspect content it can see, which makes SSL inspection a prerequisite.

  • Why is SSL inspection a prerequisite for DLP to work on file uploads to web services like dochub.com?
    Q1 — MODEL ANSWER
    File uploads to HTTPS web services are encrypted. Without SSL inspection, ZIA sees only the TCP flow metadata — it cannot read the HTTP multipart/form-data body that contains the actual file content. DLP pattern matching operates on the decrypted payload. If you exempt a file upload site from SSL inspection, DLP becomes completely blind to what's being uploaded — the HIPAA document could leave the organisation undetected.
  • What is the Confidence Score Threshold in a HIPAA DLP dictionary, and when would you need to lower it?
    Q2 — MODEL ANSWER
    The Confidence Score Threshold defines how certain ZIA must be that content matches the dictionary pattern before triggering a DLP action. A high threshold (e.g., 90%) reduces false positives but may miss real HIPAA content in documents with partial or anonymised data. If your test Medical Info PDF is not being blocked, lowering the threshold makes the engine more sensitive — it will match content that partially resembles HIPAA-pattern data. In production, tuning this value is critical to balance false positives against missed detections.
  • What happens to a legitimate business upload of a patient intake form if the DLP policy is too broad — and how would you handle this operationally?
    Q3 — MODEL ANSWER
    A broad DLP policy blocks the legitimate upload. The user sees a block page and cannot complete their workflow. Operationally, the solution is to either: (1) create a DLP exemption for specific approved destinations (e.g., the company's approved HIPAA-compliant upload portal), (2) add an allow-list for specific user groups whose role requires uploading medical data, or (3) change the action from Block to Caution + Alert so the security team is notified but the user can proceed for an authorised reason. DLP without carefully scoped exemptions creates help desk tickets, not security.

Configuration Requirements

Work through these three requirements in the ZIA Admin Portal. Check each one off as you complete it.

DLP Configuration · ZIA Admin Portal → Policy → Data Loss Prevention
  1. Log in to ZIA Admin Portal at https://admin.zscalerthree.net
  2. Navigate to Policy → Data Loss Prevention → DLP Dictionaries
  3. Search for "HIPAA" in the dictionary list
  4. Open it to review the content patterns — no changes needed, it is predefined
  1. Navigate to Policy → Data Loss Prevention → DLP Rules
  2. Click Add Rule — name it "Block HIPAA File Uploads"
  3. Set Action to Block
  4. Under Content, add the HIPAA dictionary
  5. Set the rule to apply to: all users, file upload traffic
  6. Ensure the rule is Active — click Save
  1. On the Corp: Client PC VM, open the DLP Test Files folder
  2. Open a browser and navigate to dochub.com
  3. Attempt to upload the Medical Info PDF file
  4. Confirm the upload is blocked by a ZIA block page
  5. In ZIA Admin Portal, navigate to Analytics → DLP Incident Reports to confirm the event is logged

Testing Requirements

  • Upload the Medical Info PDF to dochub.com — confirm it is blocked
  • Upload a non-sensitive file — confirm it succeeds (rule is not over-blocking)
  • Check ZIA Admin Portal → Analytics → DLP Incident Reports and confirm the block event is logged

MENTAL NOTE
ZIA DLP has three layers: (1) DLP Dictionaries (content patterns — HIPAA is predefined), (2) DLP Rules (the policy that references dictionaries and defines action + scope), (3) DLP Incident Reports (the audit trail). You must create the rule and reference the dictionary — the dictionary alone does nothing.

DLP (Data Loss Prevention)
Inspects file content in motion and blocks policy-violating transfers.
DLP Dictionary
A set of content patterns, regex, or keywords — HIPAA is built-in.
DLP Rule
The policy that says "if content matches dictionary X and action is upload, then block".
Incident Report
The per-violation log entry — shows user, file, matched dictionary, and action taken.
