Lab 1: Connect to the Virtual Lab Environment

LAB 1

Scenario: As the Zscaler administrator, you need to establish a functional, pre-configured environment (the Virtual Lab) that mirrors the Lab tenant's potential production setup — and access its core management console (Experience Center) before you can begin configuration tasks.

Objectives: Connect to Skytap; sign in to Experience Center; use VM clipboard
Time: ~10 minutes
💭 THINK FIRST · CONNECTING TO THE LAB

Before any configuration can begin, you need a working connection to both the Skytap VM environment and the Zscaler Experience Center. These are two separate systems: Skytap gives you the VMs; the Experience Center is the Zscaler management console. Understanding how credentials flow between them prevents login confusion later.

  • Your admin password for the Experience Center is the same as the VM passwords in this lab. Why is this a convenience for a lab environment but a security risk in production?
  • The VM clipboard in Skytap exists because your local machine and the VM have separate clipboard contexts. What does this tell you about the network isolation of the Skytap environment — can the VM directly reach the internet on its own?
  • You are forced to change the Experience Center admin password on first login. Why does Zscaler enforce this, and what would happen if you forgot the new password mid-lab?
Q1 — MODEL ANSWER
Sharing a single password across all accounts simplifies lab setup and reset scripts, since everything can be reset to one known value. In production this violates least-privilege — a compromised user password would also unlock admin console access, collapsing the separation between end-user credentials and administrative credentials.
Q2 — MODEL ANSWER
The VM clipboard bridge is needed precisely because the Skytap VM runs in an isolated virtual network. It cannot freely access the internet independently — that's the point of the lab: all internet traffic must be forwarded through ZCC to Zscaler. Without ZCC configured, the VM's internet access may be limited or none at all.
Q3 — MODEL ANSWER
Zscaler enforces a password change on first login to prevent labs or demos from being accessed with default vendor-set credentials, which are well-known and searchable. If you forget the new password mid-lab, you'd need to use the Zscaler emergency access account (if configured) or contact the instructor to reset the pod — there's no self-service recovery in a lab tenant.

Task 1.1: Verify Lab Access

1 Start and connect to your Skytap pod

  1. Click the Skytap Portal URL provided in your access instructions.
  2. Ensure the checkbox is checked to select all VMs.
  3. If machines have not already started, click the Play button. All VMs will launch.
All three VMs should show "Running" status. VMs start in stages — wait a few minutes for all to be ready.

Note: VMs in the environment are set to start in stages, so the whole process will take a few minutes.

Task 1.2: Sign In to the Experience Center Landing Page

Note: Throughout this lab guide, <Student FQDN> represents the domain assigned to your tenant. For example, if your admin username is student@zs9901.labtenant.com, then your Student FQDN is zs9901.labtenant.com.

1 Login to the Corp: Client PC as the local Student User

  1. Launch the Corp: Client PC from the Skytap portal.
  2. Click the Ctrl-Alt-Del icon in the Skytap Toolbar.
  3. Click the Credential (keys) icon in the Skytap Toolbar.
  4. Click the Insert Credentials icon next to the password field for the student username.
Use the Skytap Credential (keys) icon to auto-fill the student password — avoids clipboard issues at the Windows lock screen.

2 Open Experience Center and sign in

  1. In a browser window, open the Experience Center Landing Page: https://console.zscaler.com/
  2. You will be redirected to the ZIdentity login page. Authenticate with:
    • Username: student@<Student FQDN>
    • Password: as provided in the Lab Access Instructions (<Session Password>)
    • Click Sign in
  3. You will be prompted to change your password. Set a new password and remember it for all remaining labs.
Successful password reset — click Re-Login and sign in again with your new password.

Task 1.3: Use the Lab Clipboard

During these labs, you will frequently copy commands, URLs, and configuration values from the lab guide into the virtual machines.

1 Use the clipboard icon in the lab interface

  1. Click the clipboard icon in the Skytap toolbar.
  2. Paste text into the Copy/paste field. The VM clipboard will be populated.
  3. Inside the VM, use Ctrl+V (or right-click > Paste) to paste.
The VM Clipboard bridges your local clipboard with the Skytap VM. On Mac, use Cmd+V locally but Ctrl+V inside the VM.

Note: Even if you are on a Mac, the virtual machine is running Windows, so use Ctrl+V (not Cmd+V) inside the VM.

MENTAL NOTE
Three steps every lab begins with: (1) Start Skytap pod, wait for all VMs Running. (2) Log into Corp: Client PC as student. (3) Open https://console.zscaler.com/ and authenticate as student@<Student FQDN>. The Experience Center password you set in Lab 1 is used for all subsequent labs.

QUICK REFERENCE ▲
Student FQDN
Your unique tenant domain, e.g. zs9901.labtenant.com. Used in all login IDs.
Experience Center
console.zscaler.com — unified admin portal for ZIA, ZPA, ZDX, ZIdentity.
ZCC
Zscaler Client Connector — endpoint agent that tunnels all traffic to the ZTE.
Z-Tunnel 2.0
Network-layer tunnel that intercepts all IP protocols. Required for Cloud Firewall.
Service Entitlement
Grants a user access to use a Zscaler service (ZIA, ZPA, ZDX) via ZCC.
Admin Entitlement
Grants a user access to configure a service in the admin console, with a specific role.
SyncZPA
User attribute (value: Zscaler) that syncs the user into ZPA's identity store for policy use.
Forwarding Profile
Defines tunnel type/version and forwarding action per network context (on/off/VPN trusted).
Cloud App Control
Application-layer policy that identifies apps by DPI (e.g., WhatsApp) regardless of port. Separate from Firewall Filtering.
SSL Inspection
Trusted MitM: ZIA decrypts → inspects → re-encrypts using Zscaler CA. Requires Zscaler Root CA trusted on endpoint.
Cert Pinning Bypass
"Do Not Inspect" SSL rule for apps that verify server cert fingerprints (e.g., Threema). Must be above the Inspect All rule.
DLP Engine
Defines what sensitive content looks like (dictionary patterns + thresholds). Paired with a DLP Rule to define the action.
App Connector
ZPA software in the data centre that makes outbound TLS tunnels to ZPA cloud. No inbound firewall rules required.
Provisioning Key
One-time key placed at /opt/zscaler/var/provision_key. Authorises a new connector to join a specific connector group.
App Segment
ZPA object defining a private resource by domain/IP + port. Together with an Access Policy, enables ZPA access.
SIPA
Secure Internet and Private Access — routes ZPA App Connector internet traffic through ZIA for consistent egress IP + policy.
ZDX Score
0–100 composite metric of end-to-end digital experience. 0 = worst, 100 = best. <50 = Poor, 50–79 = Okay, 80+ = Good.
Y-Engine
ZDX ML engine that analyses probe data to identify root-cause factors (Wi-Fi, DNS, TCP, server response) and ranks their contribution.
ZDX Probe
Synthetic background test run by ZCC that measures DNS Time, TCP Connect, Server Response, and Page Fetch Time continuously.
