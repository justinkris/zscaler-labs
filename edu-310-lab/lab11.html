Lab 11: ZDX Copilot

LAB 11

Scenario: You want to use ZDX Copilot to answer product knowledge questions, troubleshoot device and network issues for specific users, and identify Wi-Fi root causes — all through a conversational interface.

Time: ~20 minutes
Environment: ZDX Admin Portal · Copilot (left panel icon)
💡 THINK FIRST · AI-POWERED TROUBLESHOOTING

ZDX Copilot is not just a chatbot — it has direct access to the ZDX data in your organisation. When you ask it to troubleshoot a user, it queries the live ZDX database, runs analysis, and returns findings in natural language, just as a senior engineer would.

  • Copilot identified the hop latency for Quincy’s network issue automatically — the same finding you reached manually in Lab 5. What is the operational advantage of Copilot for a Level 1 helpdesk analyst who does not have deep ZDX expertise?
  • Copilot’s responses include Sources (documentation links). Why is citing sources important for an AI assistant used in a professional IT environment?
  • Copilot uses a “frozen model” — no customer data is used to train the system, and prompts are retained for up to 14 days per data retention policy. What security and compliance implications does this have for how you should phrase queries?
Q1 — MODEL ANSWER
For a Level 1 analyst, Copilot eliminates the need to know which ZDX screen to open, which metric to examine, and what threshold indicates a problem. They can ask in plain language ("troubleshoot Barry’s device for the last 3 hours") and get an actionable finding with evidence. This dramatically reduces mean time to diagnosis (MTTD), reduces training requirements, and allows analysts to handle more complex cases without escalation.
Q2 — MODEL ANSWER
In a professional IT environment, citing sources serves several purposes: (1) Verifiability — the analyst can confirm the AI’s answer against official documentation. (2) Auditability — if an action based on Copilot output causes an issue, the cited source shows whether the recommendation was based on official guidance. (3) Trust calibration — knowing where the information comes from helps analysts judge whether it applies to their specific deployment configuration.
Q3 — MODEL ANSWER
Because prompts are retained for up to 14 days, avoid including sensitive identifiers like full email addresses, employee names in combination with health or performance data, or IP addresses of sensitive systems. Phrase queries generically where possible (e.g. "troubleshoot Barry" rather than "Barry Bogdown, employee ID 12345"). Also be aware that prompts are stored in Zscaler logs — ensure your queries comply with your organisation’s data handling and privacy policies.

Task 11.1: Using Copilot for Product Knowledge Questions

1 Ask a Product Knowledge Question

  1. In the ZDX Admin Portal, click the Copilot icon in the left panel.
  2. In the Copilot page, type the following question and press Enter:
    • “How do I configure ZDX Hosted Monitoring?”
  3. Review the answer and the Sources references (linked documentation pages).
  4. Try additional suggested questions at the bottom of the response (e.g. “What is ZDX?”, “What are the benefits of using ZDX?”).

Task 11.2: Understanding Customer Local or Upstream Network Issues Using Copilot

2 Troubleshoot a Device Issue with Copilot

  1. Click + New Session in the Copilot sidebar to start a fresh conversation.
  2. Enter the following query:
    • “Troubleshoot device issues for Barry in the last 3 hours”
  3. Review Copilot’s response including Device Issues for Barry Bogdown and Application Issues.
  4. Navigate to Device Performance › Top Processes By CPU Incidents as directed to verify the CPU usage by Adobe Premiere Pro.exe.
  5. Create another new session and enter:
    • “Troubleshoot network issues for Quincy in the last 3 hours”
  6. Review the Copilot response showing the specific high-latency hops identified automatically.

Task 11.3: Wi-Fi Issue Using Copilot

3 Diagnose a Wi-Fi Issue Conversationally

  1. Create a new Copilot session and input:
    • “Troubleshoot the network issue for Curtis for the last 4 hours.”
  2. Observe the root cause identified by Copilot as Low Wi-Fi Signal.
  3. Navigate to Device Performance › Wi-Fi Signal to justify the root cause identified by Copilot.

Result: Copilot identifies Wi-Fi performance issues due to change of Wi-Fi type to 802.11n, SSID to Columbus-OH-Office, Channel 1, and Signal Strength to 15.0%. The Device Performance Wi-Fi Signal graph confirms the low signal strength.
