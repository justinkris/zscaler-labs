Understanding Wi-Fi Dashboard

The Wi-Fi Dashboard is ZDX’s dedicated view for monitoring Wi-Fi access point performance across your organisation. It aggregates per-device Wi-Fi data to give you an access-point level view — identifying which specific APs are causing poor user experience.

Lab 14: Understanding Wi-Fi Dashboard

LAB 14

Scenario: You want to proactively monitor Wi-Fi Access Point performance across your organisation. Use the ZDX Wi-Fi Dashboard to identify poor-performing APs, review connected devices, and examine Source to Gateway Latency.

Time: ~15 minutes
Environment: ZDX Admin Portal · Dashboard › Wi-Fi Dashboard
💡 THINK FIRST · WI-FI DASHBOARD

Individual user Wi-Fi problems (Labs 2 and 5) are reactive. The Wi-Fi Dashboard is proactive: it aggregates Wi-Fi data from all devices to give you an access-point level health view. One red AP on the dashboard = every user connected to that AP is potentially affected.

  • The Wi-Fi Dashboard shows access point performance at the AP level, while the User Overview shows Wi-Fi data per user. What is the operational advantage of the AP-level view for operations staff?
  • AP-Columbus-OH shows as Poor (red) with a Source to Gateway Latency of 144.4ms average. Curtis Hardwick is in the Active Users table for this AP with a ZDX Score. How does the Wi-Fi Dashboard accelerate the root cause investigation compared to starting from the User Overview?
  • The Wi-Fi Band Distribution shows 90% of devices on 5GHz and 40% on 2.4GHz (some devices have both). If the dashboard shows an AP is Poor but most connected devices are on 2.4GHz, what does this suggest about the root cause?
Q1 — MODEL ANSWER
The AP-level view allows operations staff to identify problems that affect multiple users simultaneously without needing to review individual user reports. If one AP is red, the operations team can dispatch a site engineer to physically inspect that AP, reset it, or replace it — without having to correlate multiple user complaints. It also reveals AP health trends over time: is this AP always poor, or is it a new issue?
Q2 — MODEL ANSWER
Wi-Fi Dashboard acceleration: (1) You immediately see that AP-Columbus-OH is the shared point of failure, rather than wondering if multiple users have individual issues. (2) The Active Users table shows all affected users directly — you can see Curtis Hardwick and his ZDX Score without searching. (3) Source to Gateway Latency confirms the AP-level measurement, not just the user’s perception. You go from "user complaint" to "specific AP with high latency" in one screen.
Q3 — MODEL ANSWER
2.4GHz is more susceptible to interference from neighbouring networks, microwaves, and Bluetooth devices because it has only 3 non-overlapping channels (1, 6, 11). If most connected devices are on 2.4GHz and the AP is Poor, the root cause is likely channel congestion or interference at 2.4GHz. Recommended action: check neighbouring APs and consumer devices for channel overlap, and consider configuring band steering to push capable devices to 5GHz.

Task 14.1: Accessing Wi-Fi Dashboard

1 Explore the Wi-Fi Dashboard

  1. From the ZDX Admin Portal, navigate to Dashboard › Wi-Fi Dashboard.
  2. Review the summary metrics at the top:
    • Total Access Points
    • Wi-Fi Performance of Access Points (Good/Poor bar)
    • Total Devices Connected
    • ZDX Score Distribution (Good/Okay/Poor)
  3. Review the Wi-Fi Band Distribution pie chart (5GHz vs. 2.4GHz).
  4. Review the Wi-Fi Performance by Geolocation world map.
  5. Mouse-over or click a map marker to see access points and devices for that region.
  6. Click View Access Points to see the Wi-Fi Access Points list. Toggle between Tree View and List View.
  7. Click View Details for an Access Point shown in Poor (red) category.
  8. Review the AP Details: SSID, Location, AP MAC, Wi-Fi Performance (Good/Poor), Devices connected.
  9. Scroll down to Key Metrics → Source to Gateway Latency and review the latency graph.
  10. Scroll to the Active Users section and examine the connected devices table (Device, User, BSSID, Wi-Fi Adapter, ZDX Score, Signal Strength, Wi-Fi Type, Channel, OS).

QUICK REFERENCE ▲
Wi-Fi Dashboard
ZDX dashboard aggregating Wi-Fi performance data at the Access Point level. Shows AP health, connected devices, and Source to Gateway Latency.
Source to Gateway Latency
Roundtrip latency measured from the client device to the Wi-Fi gateway. High values indicate AP congestion or interference.
Wi-Fi Band Distribution
Breakdown of connected devices by Wi-Fi frequency band (5GHz vs. 2.4GHz). Shown as a pie chart on the Wi-Fi Dashboard.
AP Performance (Good/Poor)
Binary classification of each access point based on the ZDX Scores of devices connected to it. Poor = majority of connected devices have Poor ZDX Scores.
Active Users Table
Per-AP table in Wi-Fi Access Point Details showing each connected device with BSSID, ZDX Score, Signal Strength, Wi-Fi Type, Channel, and OS.
View Access Points
Link from the Wi-Fi Performance by Geolocation map popup. Opens the Wi-Fi Access Points list for that location in Tree or List View.
