💡 THINK FIRST · ZDX DASHBOARD ORIENTATION

A skilled ZDX operator knows which dashboard to open before they know what the problem is. The portal organises data by scope: fleet-wide incidents, per-application trends, per-user drill-downs.

  • The Performance Overview shows Most Impacted Applications and a ZDX Score map. What is the ZDX Score and how is it calculated from 0–100?
  • ZDX has Performance Overview, Incidents Overview, and Self Service Overview dashboards. When would you start at Incidents instead of Performance?
  • If the Y-Engine reports Low Wifi Signal with 90% confidence but the user insists their Wifi is fine, what would you check next to validate or disprove the finding?
Q1 — MODEL ANSWER
The ZDX Score is a composite 0–100 metric derived from Web Probe measurements (Page Fetch Time, Server Response Time, DNS Time, Availability) and Cloud Path latency. Higher is better. The score normalises raw measurements against a per-user baseline so that a user with a slow connection scores fairly against one with a fast connection.
Q2 — MODEL ANSWER
Start at Incidents Overview when a user reports an issue that seems to affect a location or subnet, not just their own device. Incidents track events affecting multiple users simultaneously — Wi-Fi degradation, Last Mile ISP problems, Zscaler Data Center issues. If only one user is impacted, go to Performance Overview and drill into Users.
Q3 — MODEL ANSWER
Check the Cloud Path Hop View at the same timestamp. A Wifi issue would manifest as high latency on the first hop between the client and its gateway. Also check User Device Events — the Network interface tab will show SSID changes or Wifi signal strength drops at the exact moment the score dropped.

Task 1.1: View ZDX Dashboard

1 Log In and Explore Performance Overview

  1. Launch your browser and navigate to https://admin.zdxcloud.net/ or click Open ZDX Admin Portal in the Skytap frame.
  2. Log in with your ZDX Admin credentials:
    • Admin User ID: ptcb76-ADMIN@thezerotrustexchange.com
    • Session Password: <Session Password>
  3. At the top of the Performance Overview page, select 24 Hours from the dropdown.
  4. Scroll through Most Impacted Applications — click any app to update the ZDX Score map.
  5. Use Draw Fence to select a geographic area, then click Filter Selection.
  6. Click Geolocations, select a location, and click Apply. Then click Reset.
  7. Go to Dashboard › Incidents Overview and review Total Incidents and Incidents Across Key Areas.
  8. Scroll to the Incidents by Epicenters map and click a marker to view incident details.
  9. Go to Dashboard › Self Service Overview and review notification summary graphs.

Note: You are accessing the ZDX Admin Portal as a Read-Only Administrator. You can view all dashboards and reports but cannot save configuration changes.

Task 1.2: View Applications Overview Dashboard

2 Browse Application Performance

  1. From the left menu, select Applications.
  2. Sort the table by ZDX Score Trend and click the application with the lowest ZDX Score.
  3. In the ZDX Score Over Time graph, drag to zoom in on a period of significant change.
  4. Click Zoom Out to reset the view.
  5. Scroll to the Impacted Departments, Impacted Regions, and Impacted Zscaler Locations panels.
  6. View the application’s configured Probe Status panel.

Task 1.3: View User Overview Dashboard

3 Browse User-Level Performance

  1. From the left menu, select Users.
  2. Select an application from the dropdown and click Apply.
  3. Under ZDX Score User Distribution, toggle between Poor, Okay, and Good to update the user table.
  4. Select a user with Poor or Okay performance.
  5. Click View More Device Details and explore the Hardware, Network, and Software tabs.
  6. Scroll to the Web Probe Metrics section and click a point on the Page Fetch Time graph to see the breakdown.
  7. Scroll down to Cloud Path metrics and explore the Hop View and Command Line View.
  8. Scroll to Device Health and click a point on the CPU Usage graph to see Most Impacted Processes.
  9. Examine the User Device Events timeline.

Task 1.4: View Automated Root Cause Analysis Data

4 Use the ZDX Y-Engine

  1. From the left menu, select Search and type a user name (e.g. Curtis Hardwick).
  2. Click the user name in the search results to open their report.
  3. Select an application and click Apply.
  4. In the ZDX Score Over Time graph, click a point with a Poor score and click Analyze Score.
  5. Review the The following factors that might have impacted the ZDX score table, including Factor, Explanation, and Confidence Level.
  6. Drag to select a time range and click Analyze Range. Review the results.
  7. Click Compare to and select Same time 1 day ago. Scroll through the side-by-side comparison.

LAB 1 PROGRESS
QUICK REFERENCE ▲
ZDX Score
0–100 composite metric derived from Web Probe and Cloud Path data, normalised to a per-user baseline.
Y-Engine
ZDX AI/ML engine that analyses a score point and lists probable root-cause factors with confidence percentages.
Web Probe
Synthetic probe measuring Page Fetch Time, Server Response Time, DNS Time, and Availability from the user’s device to the application.
Cloud Path Probe
ICMP traceroute-style probe measuring hop-by-hop latency from the client to the application server.
Deep Tracing
On-demand diagnostic session capturing detailed device health and network path data for a specific user and time window.
ZDX Score User Distribution
Pie chart on the User Overview page categorising users into Poor, Okay, and Good score buckets for a selected application.
