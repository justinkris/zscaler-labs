Lab 10: Sharing ZDX Snapshots

LAB 10

Scenario: You need to share a ZDX user performance report with a manager who does not have ZDX portal access. Create a shareable snapshot URL, configure privacy settings, and then manage the snapshot in the portal.

Time: ~15 minutes
Environment: ZDX Admin Portal · Self Service Dashboard → Share Snapshot → Analytics › ZDX Snapshots
💡 THINK FIRST · ZDX SNAPSHOT SHARING

ZDX Snapshots solve a real operational problem: how do you share a complex ZDX report with someone who doesn’t have portal access? The snapshot is a read-only URL, valid for a defined time, with optional data obfuscation for privacy.

  • A ZDX Snapshot URL is accessible by anyone with the link, even outside your organisation. In what business scenarios would you choose to enable Obfuscate Data, and what does obfuscation actually hide?
  • You can set a snapshot validity period (2 hours, 1 day, etc.). What factors would guide your choice of validity period for different use cases?
  • The ZDX Snapshots management page shows a View Count column. What operational value does this metric provide to an administrator managing shared snapshots?
Q1 — MODEL ANSWER
Enable Obfuscate Data when sharing externally (with a vendor, MSP, or user) where you do not want to expose usernames, email addresses, or device names. Obfuscation replaces identifying information with anonymised tokens. You would skip obfuscation when sharing internally with a manager or IT team who needs to see the specific user and device names to take action.
Q2 — MODEL ANSWER
Factors: (1) Urgency of the issue — for an active outage being shared with vendor support, 2 hours is sufficient. (2) Compliance requirements — some organisations have data retention policies that require minimising exposure time. (3) Expected response time — if you’re sharing with a QBR stakeholder who may review it over a week, choose a longer validity. (4) Sensitivity of the data — shorter validity for snapshots containing identifiable user data.
Q3 — MODEL ANSWER
View Count tells the administrator whether the snapshot was actually accessed. A View Count of 0 after several hours means the recipient may not have seen it — you may need to resend or follow up. A high View Count may indicate the URL was forwarded beyond the intended recipient. It provides a lightweight audit trail for snapshot access without requiring a full access log.

Task 10.1: Prepare User Report

1 Open a User Report to Share

  1. In the ZDX Admin Portal, click Dashboard › Self Service Dashboard.
  2. In the Notifications Sent table at the bottom, click on a Username to open their user report.
Self Service Dashboard with a Username in the Notifications Sent table selected — clicking it opens the user’s ZDX report.

Task 10.2: Generate Report Snapshot & Copy Snapshot URL

2 Create a Shareable ZDX Snapshot

  1. Scroll to the top of the user report page and click Share Snapshot.
  2. In the Share ZDX Snapshot dialog, enter:
    • Name: Student <Student ID> (replace with your assigned Student ID)
    • Valid for: 2 Hours
    • Obfuscate Data: Checked (green)
  3. Click Next.
  4. Click Copy URL to copy the snapshot URL to your clipboard.
  5. Click Close.
  6. Optionally: open a new browser tab and paste the URL to view the snapshot.

Warning: ZDX Snapshots can be accessed by anyone with the link, even outside your organisation. Always confirm whom you are sharing with before sending the URL.

Task 10.3: Manage ZDX Snapshots

3 View and Delete Snapshots

  1. In the ZDX Admin Portal, go to Analytics › ZDX Snapshots.
  2. Use the Search bar to find the snapshot you just created.
  3. Review the Status column (Active, Initiated, Expired, Snoozed).
  4. Click the Trash icon to delete the snapshot you created.
ZDX Snapshots management page showing snapshot list with Name, Created On, Valid Until, Status, URL, View Count, and delete Actions.

MENTAL NOTE
ZDX Snapshot = read-only URL of a ZDX report, no login required. Configure: Name, Valid For (2hrs to indefinite), Obfuscate Data (hides usernames/device names). Copy URL → share. Manage: Analytics › ZDX Snapshots → search, check Status (Active/Expired/Snoozed), View Count, delete. Security: anyone with the link can view it — use Obfuscate Data for external sharing.

ZDX Snapshot
View-only URL capturing the current state of a ZDX report. Accessible without portal login. Valid for a configurable duration.
Obfuscate Data
Snapshot option that anonymises identifying information (usernames, device names, email addresses) while preserving all metrics and graphs.
Snapshot Status
Active = accessible. Initiated = being generated. Expired = validity period has passed. Snoozed = temporarily paused.
View Count
Number of times the snapshot URL has been opened. Provides a lightweight audit trail for snapshot access.

Snapshot management page. Search, view status, check View Count, and delete snapshots from here.
