💡 THINK FIRST · DNS ROOT CAUSE

DNS is the phone book of the internet. When it’s slow, every SaaS app is slow — but real-time protocols like Zoom calls (which use pre-resolved IPs) are unaffected. This asymmetry is the classic diagnostic tell.

  • Linda says Zoom calls are fine but web-based SaaS apps are slow. Before looking at ZDX data, what does this symptom pattern already tell you about where the problem is?
  • The ZDX Score is consistently Poor (not oscillating) across every application for 48 hours. What class of root cause does this pattern most strongly suggest?
  • Y-Engine identifies DNS Server 10.0.1.53 as the culprit. What actions would you take next, and who would you escalate to?
Q1 — MODEL ANSWER
Zoom uses pre-negotiated IP addresses and persists a connection once established — so it bypasses DNS lookups for the duration of the call. Web-based apps issue a new DNS query on every page load. Slow DNS means every SaaS page load is bottlenecked before the first byte is even requested from the server. This pattern points directly at DNS.
Q2 — MODEL ANSWER
A persistently low score across all applications for 48 hours points to a infrastructure-layer problem, not a single app or device issue. The most common candidates are: (1) DNS server failure or misconfiguration, (2) a misconfigured network route or ISP issue, (3) a failing ZCC tunnel. The fact that all apps are affected equally rules out an app-specific issue.
Q3 — MODEL ANSWER
Escalate to the network team with the specific DNS server IP (10.0.1.53). Ask them to: (1) check the DNS server’s response times and query logs, (2) verify it is not overloaded or misconfigured, (3) consider failing over to a secondary DNS server (8.8.8.8 or a corporate DNS with correct forwarders). After the fix, monitor Linda’s ZDX Score for recovery — it should return to Good within one probe cycle.

High DNS
Y-Engine factor triggered when DNS resolution time is the primary driver of elevated Page Fetch Time.
DNS Resolve Time
Web Probe metric measuring the time from DNS query to resolution. Shown as a sub-component of Page Fetch Time.
PFT Breakdown
Page Fetch Time is the sum of: DNS Resolve Time + TCP Connect Time + SSL Handshake + Server Response Time (TTFB) + Content Download.
Consistently Poor (all apps)
When all applications show Poor ZDX Scores simultaneously, the root cause is almost always infrastructure: DNS, routing, or ZCC tunnel — not application-specific.
