Lab 13: Understanding Data Explorer

LAB 13

Scenario: You want to investigate whether a DNS performance problem is affecting multiple SaaS applications simultaneously, and see which geolocations are experiencing the worst ZDX Scores. Build a custom Data Explorer view to find out.

Time: ~15 minutes
Environment: ZDX Admin Portal · Analytics › Data Explorer
💡 THINK FIRST · CUSTOM REPORTING

Pre-built dashboards answer the questions Zscaler anticipated. Data Explorer answers the questions your organisation generates: “Does Outlook and ServiceNow both suffer when DNS is slow?” or “Which geolocations have the worst ZDX Score right now?”

  • Data Explorer shows you can compare ZDX Score and DNS Time for multiple applications simultaneously. You observe that when DNS Time spikes, both Outlook and ServiceNow ZDX Scores drop at the same time. What does this co-occurrence prove, and what does it rule out?
  • Data Explorer supports two view types: chart (line graph) and tabular. When would you use each view, and what insights does the tabular view provide that the chart does not?
  • You can Group By Applications, Geolocations, Departments, User Groups, and Zscaler Locations. You are investigating whether a branch office has worse performance than headquarters. Which Group By would you use, and which metric would you prioritise?
Q1 — MODEL ANSWER
Co-occurrence proves the root cause is shared between both applications. Since Outlook and ServiceNow use different servers and CDNs, the only thing they share is the user’s DNS resolver. The simultaneous spike in DNS Time for both applications rules out: (1) an application-specific problem, (2) a network path problem to one specific server, and (3) a device CPU problem (which would not affect DNS Time directly). It strongly implicates the DNS resolver.
Q2 — MODEL ANSWER
Chart view is best for identifying temporal patterns: when does a metric spike, how long does it persist, does it correlate with another metric? Tabular view is best for comparative analysis across many entities at a single point in time: which geolocations have the worst score right now, or which department has the highest average DNS Time. The table gives you exact numbers you can sort and export, while the chart gives you trend and correlation.
Q3 — MODEL ANSWER
Use Group By: Geolocations (to see per-location data). Prioritise ZDX Score (the composite health metric) as the primary filter, then drill into Page Fetch Time and DNS Time for locations with poor ZDX Scores. If you also want to isolate Zscaler infrastructure effects, add a second view grouped by Zscaler Locations to see if certain ZIA nodes correlate with poor scores in specific geolocations.

Task 13.1: Accessing Data Explorer

1 Build and Customise a Data Explorer View

  1. Navigate to Analytics › Data Explorer.
  2. View the list of existing reports. Click the eye icon on the Microsoft Teams Call Quality report to view it.
  3. Click Edit to modify the report. Configure the following:
    • Data Source: End User Monitoring
    • Applications: SharePoint Online and Outlook Online
    • Metrics: ZDX Score, Page Fetch Time, and Packet Loss
    • Group By: Applications
    • Aggregation Type: Average
  4. Click Run Query and review the Average ZDX Score and Average Page Fetch Time graphs side by side.
  5. Change the Metrics to ZDX Score and DNS Time, then click Run Query. Observe the correlation between DNS Time and ZDX Score.
  6. Replace Outlook Online with ServiceNow in the Applications list and run the query again. Observe that high DNS Time impacts both Outlook and ServiceNow ZDX Scores.
  7. Change the Group By to Geolocations and switch to the tabular view. Review the Average ZDX Score per geolocation.

LAB 13 PROGRESS
QUICK REFERENCE ▲
Data Explorer
ZDX custom reporting engine. Build views with configurable Data Source, Applications, Metrics, Group By, Aggregation Type, and View Type.
Aggregation Type
How metric values are combined: Average, Minimum, or Maximum. Average is most common for trend analysis.
Group By
Dimension by which data is segmented: Applications, Geolocations, Departments, User Groups, or Zscaler Locations.
End User Monitoring
Data source option in Data Explorer for user-measured ZDX data from Zscaler Client Connector.
Hosted Monitoring
Data source option in Data Explorer for probe-measured data from Zscaler Hosted Probes.
Tabular View
Data Explorer display mode showing metric values as a sortable table, grouped by the selected Group By dimension.
