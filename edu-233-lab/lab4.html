Lab 4: Using Isolate Action in URL Filtering Policies

LAB 4

Scenario: The Isolate action in URL Filtering policies extends the classic Allow/Block binary. Traffic matching a configured URL category is rendered in an isolated browser using the selected Isolation Profile — giving users productive access to sites like news and social media without exposing the endpoint to their risk.

Time: ~20 minutes
💡 THINK FIRST · URL POLICY ISOLATION

Isolate as an action in URL Filtering is the primary tool for category-based Browser Isolation. It removes the false choice between "block and frustrate users" or "allow and accept risk" for entire URL categories.

  • A URL Filtering policy with Isolate action is configured for "News and Media." A user navigates to bbc.com. Describe the full traffic flow from endpoint to isolated browser.
  • The "Isolate News Media" policy uses the Copy-Print-Watermark profile. What specific data exfiltration paths does this profile allow, and which does it block?
  • If a user in an isolated news session tries to paste text from their local clipboard into the page, what happens — and why?
Q1 — MODEL ANSWER
ZCC forwards the HTTPS request to ZIA. ZIA decrypts the traffic (SSL inspection), evaluates URL Filtering policies, matches "Isolate News Media," and instead of returning the page content directly, ZIA spins up an isolated container in the configured region. The container fetches bbc.com, renders it, and streams a pixel-stream back to the user's browser. The user interacts with the stream; all requests originate from the container, not the endpoint.
Q2 — MODEL ANSWER
Copy-Print-Watermark allows: (1) copying content from the isolated page to the local clipboard ("Isolation to local computer"), and (2) printing from the isolated session. It blocks: uploading files to the isolated page and downloading files from it to the local machine. Additionally, the session is watermarked with the user's identity and timestamp.
Q3 — MODEL ANSWER
Pasting from the local clipboard to the isolated page is blocked because "Copy from Local Computer to Isolation" is disabled in the Copy-Print-Watermark profile. The clipboard bridge from the user's OS to the isolation container is severed, preventing data exfiltration into the remote site.

Task 4.1: Review URL Filtering Policies with Isolate as Action

1 Inspect the Isolate Policies

  1. In the ZIA Admin Portal, go to Policy › URL & Cloud Application Control › URL Filtering Policy.
  2. Scroll through the policy list and locate the Isolate News Media and Isolate Sports rules.
  3. Click the eye icon next to Isolate News Media to review its settings.
  4. Note the URL Categories (News and Media), Action: Isolate, and Isolation Profile: Copy-Print-Watermark.
  5. Close the rule viewer.

Task 4.2: Test Isolate as an Action in URL Filtering Policies

1 Access a News Site in Isolation

  1. On the Client PC VM, open a browser and navigate to https://www.bbc.com.
  2. Observe that bbc.com loads in isolation with the isolation banner.
  3. The Copy-Print-Watermark profile is active — you can copy from the page and print, and the session is watermarked.
  4. Try to paste text from your local clipboard into the isolated page — confirm you cannot (copy from local to isolation is disabled).

Note: You might be prompted to grant clipboard access. Click Allow if prompted — this enables the permitted direction (isolation → local), not the blocked direction (local → isolation).

MENTAL NOTE
The Isolate action in URL Filtering turns any URL category into a "safe browsing" zone. The Isolation Profile determines the exact controls — Copy-Print-Watermark allows outbound content flow (from isolation to local) but blocks inbound (from local to isolation), preventing data exfiltration into the remote container.
