Lab 8: Protecting Against Zero Day Threats with Cloud Sandbox + Isolation

LAB 8

Scenario: Zscaler's Cloud Sandbox integrates with Browser Isolation via the "Quarantine and Isolate" action. When a user downloads a file that Sandbox has never seen before (first-time file), it is instantly rendered in an isolated container as a flattened PDF. The user can review the content immediately while Sandbox analyses the original file. If the verdict is benign, the original can be downloaded. If malicious, only the flattened PDF is available.

Time: ~30 minutes
💡 THINK FIRST · SANDBOX + ISOLATION INTEGRATION

Sandbox + Isolation solves a classic security-vs-productivity tradeoff. Instead of "block and wait" (frustrating) or "allow and hope" (risky), users get immediate safe access to content while Sandbox analyses the original — parallel processing of safety and productivity.

  • The Sandbox policy uses "Quarantine and Isolate" as the First-Time Action. What does "first-time" mean in this context, and why is this a practical approach rather than analysing every downloaded file?
  • When a potentially malicious file is in Sandbox analysis, the user sees it in Protected Storage with a "semi-closed circle" status icon. What exactly is the user viewing at this point, and is it the original file?
  • The Sandbox Detail Report shows a MITRE ATT&CK mapping. Why is this level of detail valuable to a security analyst rather than just a benign/malicious verdict?
Q1 — MODEL ANSWER
"First-time" means Zscaler has never seen this specific file hash (MD5) before — it has no prior analysis record. Sandbox uses this to prioritise analysis resources: known-good files (seen many times, already verdicted benign) pass through immediately. Only truly novel files are quarantined, which keeps the system scalable without degrading performance for routine downloads.
Q2 — MODEL ANSWER
The user is viewing a flattened PDF rendered from the original file inside the isolation container. It is NOT the original file. The PDFTron engine converted the file to PDF, stripping all macros and active content. The original file is held by Sandbox for analysis. Even if the file is malicious, the PDF rendering is safe — the endpoint never receives executable content.
Q3 — MODEL ANSWER
A MITRE ATT&CK mapping tells the analyst exactly which techniques the file attempted to use (e.g., T1566.001 Spearphishing Attachment, T1059.001 PowerShell). This enables precise detection rule updates, IR playbook selection, and reporting to leadership using the industry-standard threat taxonomy. A binary benign/malicious verdict only tells you the outcome — the ATT&CK map tells you the method.

Task 8.1: Review the Cloud Sandbox + Isolation Configuration

1 Verify the Sandbox-Enabled Isolation Profile

  1. In the ZIA Admin Portal, navigate to Administration › Browser Isolation.
  2. In the list, locate the TZTE Download and Copy profile.
  3. Confirm that File Transfer From: Isolation to local computer is set to Sandbox Scanned.
  4. This means files can only be downloaded from the isolation container to the local machine after Sandbox has cleared them.

Note: The "Sandbox Scanned" setting requires an Advanced Sandbox subscription. With a Read-Only Admin account you will see the profile but cannot edit it.

2 Review the Sandbox Isolation Policy

  1. Navigate to Policy › Sandbox.
  2. Click the eye icon next to the Sandbox Isolation policy.
  3. Review the File Types selected (Excel, PowerPoint, Word, PDF, RTF).
  4. Note the First-Time Action: Quarantine and Isolate and Isolation Profile: TZTE Download and Copy.
  5. Close the policy viewer.

Task 8.2: Test the Sandbox + Isolation Configuration

1 Trigger a Sandbox + Isolation Event

  1. On the Client PC VM, navigate to https://securitytest.zsdemo.com/passwordless/browser-isolation.php.
  2. The page opens in Browser Isolation with the banner: "Sandbox analysis is in progress."
  3. Note the file download appears in the browser with a "Sandbox Analysis in progress" (semi-closed circle) status icon.
  4. Click the Zscaler icon in the bottom-right corner to access the isolation menu, then click Show saved files.
  5. In Protected Storage, observe the file with the spinner icon indicating ongoing Sandbox analysis.

2 Review the Sandbox Verdict and Logs

  1. Wait for the banner: "Sandbox Analysis is completed and the file was found to be safe."
  2. Once verdicted safe, you can download the original file from Protected Storage.
  3. Navigate to Analytics › Web Insights › Logs.
  4. Add filters: User = your MKT username, URL Search (Host Contains) = securitytest.zsdemo.com.
  5. Click Apply Filters. In the column picker (3 dots), add the MD5 column.
  6. Click the MD5 hash link in the results to open the Sandbox Detail Report.
  7. Review the Classification, MITRE ATT&CK mapping, and Virus and Malware sections.
