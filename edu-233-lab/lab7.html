Lab 7: Applying Document Isolation Controls

LAB 7

Scenario: Browser Isolation can intercept file downloads and render them as PDFs inside the isolated container, preventing malicious files from ever reaching the endpoint. The isolated container's Protected Storage holds downloaded files for the duration of the session. When the session ends, Protected Storage is purged.

Time: ~25 minutes
💡 THINK FIRST · DOCUMENT CONTROLS IN ISOLATION

Document isolation converts every downloaded file into a flattened PDF before the endpoint ever sees it. This eliminates macro-based threats, embedded scripts, and zero-day exploits in Office documents — without blocking file access entirely.

  • The "Print Only" Isolation Profile has "Allow Viewing Office Files" enabled. When a user downloads a .docx from an isolated session, what happens to that file before they can read it?
    Q1 — MODEL ANSWER
    The .docx is downloaded to Protected Storage inside the isolated container. Zscaler uses PDFTron (an embedded PDF engine) to convert the file into a flattened PDF. The user views the PDF rendering — not the original .docx. This process strips all macros, active content, embedded scripts, and OLE objects before the user ever interacts with the content.
  • Protected Storage is described as temporary. What is the security model behind purging Protected Storage when the isolation session ends?
    Q2 — MODEL ANSWER
    Protected Storage is isolated to a single session. When the session ends (browser tab closed, session timeout), the storage is wiped. This prevents data from one session being accessible to the next — particularly important in shared device scenarios or BYO environments where a second user might open an isolated session on the same machine.
  • A user needs to download a file from an isolated session to their local machine. The isolation profile's "File Transfer From: Isolation to local computer" is disabled. What options does the user have?
    Q3 — MODEL ANSWER
    If file download to local is blocked by the profile, the user can (1) view the PDF rendering in isolation and manually transcribe needed content, (2) request an admin to adjust the profile to allow Flattened PDF downloads (not the original file), or (3) escalate for an exception. The admin should not enable "Original File" download for risky content — flattened PDF is the secure middle ground.

Task 7.1: Review Document Viewing Settings in the Isolation Profile

1 Inspect the Print Only Profile Security Settings

  1. In the ZIA Admin Portal, navigate to Administration › Browser Isolation.
  2. In the list of Isolation Profiles, click the eye icon next to Print Only.
  3. On the Security tab, scroll down to review:
    • Restrict Text Input: Read-Only Isolation — Enabled
    • Allow Viewing Office Files: View office files in isolation — Enabled
    • Local Browser Rendering: Disabled
  5. With this profile: clipboard/copy/paste is blocked, page is read-only and watermarked, Office file viewing is enabled, and printing is allowed.
  6. Click Close.

Task 7.2: Test Document Viewing in Isolation

1 Download and Render a Document in Isolation

  1. On the Client PC VM, navigate to https://file-examples.com/index.php/sample-documents-download/sample-doc-download/.
  2. Verify that the site opens in isolation (check for the isolation banner).
  3. Click DOWNLOAD SAMPLE DOC FILE for the 100kB DOC file.
  4. The file downloads to the Protected Storage of the isolation session.
  5. Click the downloaded file in the browser's download bar to have it rendered as a PDF inside the isolated session.
  6. Review the PDF rendering — note it is a flattened representation of the .doc file.

2 Check Protected Storage and Print

  1. Click the Zscaler icon in the bottom-right corner of the isolated session.
  2. Click the document icon and select Show saved files to open Protected Storage.
  3. Confirm the .doc file appears in Protected Storage.
  4. Attempt to print the file from within the isolated session.

Note: Isolation can convert many document formats to PDF for viewing: .doc, .docx, .xls, .xlsx, .ppt, .pptx, .pdf, .rtf, and more. Files are never written to the endpoint disk during an isolated session.

MENTAL NOTE
