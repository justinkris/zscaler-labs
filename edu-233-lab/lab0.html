Lab 0: Connecting to the Virtual Lab

LAB 0

Scenario: Before any Browser Isolation policy can fire, your virtual machine must be running and Zscaler Client Connector must be authenticated. In this lab you will start the Client PC VM, log in to ZCC as the Marketing department test user, and confirm that web traffic is forwarded through the Zscaler cloud.

Time: ~15 minutes
💡 THINK FIRST · ZCC CONNECTIVITY

Browser Isolation is enforced cloud-side by the Zscaler proxy — it cannot act on traffic that ZCC never forwards. Every BI lab in this guide depends on ZCC being authenticated and active before you start.

  • ZCC tunnels device traffic to ZIA before any policy is evaluated. What happens to Browser Isolation policies if ZCC is disabled or unenrolled on the endpoint?
  • The lab uses a Marketing (MKT) department test user. Why does department assignment matter for the BI policies you will test in later labs?
  • If ip.zscaler.com confirms your traffic is routing through Zscaler, but the ZIA portal shows the user as Unauthenticated, what would you check first?
Q1 — MODEL ANSWER
BI policies never fire. Without ZCC (or a PAC file / GRE tunnel) forwarding traffic, the Zero Trust Exchange never sees the session. BI is enforced inside the Zscaler proxy on already-forwarded traffic — the endpoint itself is not modified.
Q2 — MODEL ANSWER
ZIA URL filtering and Cloud App Control policies can be scoped by department. The pre-built BI policies in this lab guide target the MKT department specifically. Using a different department user would mean those rules would not match and isolation would not trigger.
Q3 — MODEL ANSWER
Check ZCC's Authentication Status first. If it shows Unauthenticated, the SAML redirect may have failed or the session token expired. Verify the Session Password is correct and confirm you dismissed the 'Stay signed In?' prompt with No — accepting it can cache stale credentials.

Task 0.1: Start the Lab Environment

1 Start the Skytap Pod

  1. Navigate to the Skytap Access URL provided in your lab confirmation email.
  2. Ensure all VMs are selected and click Play to start them.
  3. Wait for the Corp: Client PC VM to reach Running status before continuing.

Note: VMs start in stages — allow a few minutes for all machines to be ready before proceeding.

Task 0.2: Log In to Zscaler Client Connector

1 Authenticate ZCC as the Marketing Test User

  1. On the Client PC VM, open Zscaler Client Connector from the system tray.
  2. Enter the username: <prefix>-MKT@thezerotrustexchange.com and click Login, then Next.
  3. Enter the Session Password: <Session Password> and click Sign in.

  4. Caution: At the Stay signed In? prompt, click No.

  5. ZCC minimises to the system tray. Right-click the Zscaler icon and select Open Zscaler.
  6. Confirm: Private Access: ON, Internet Security: ON, Authentication Status: Authenticated, Network Type: Off-Trusted Network.

Task 0.3: Verify Traffic Routing Through Zscaler

1 Confirm the Zscaler Cloud is Active

  1. Open a browser window and navigate to http://ip.zscaler.com.
  2. Confirm the page reads: "You are accessing the Internet via Zscaler Cloud."
  3. Note the IP address displayed — this is a Zscaler data centre IP, not your own.
  4. In a new tab, navigate to https://www.ipaddress.my and look up the IP. It should resolve to Zscaler Inc.
