LAB 7

Scenario: Zscaler's inline cloud sandbox analyzes unknown files in real time using AI-driven techniques, quarantining suspicious downloads before they reach the user. This lab configures sandbox policies, sets up real-time malware alerts, and reviews the Sandbox Activity Report.

🤔 Think First — Before You Click 3 questions to prime your thinking ▾

A truly novel malicious file has no prior reputation record — blocking would prevent benign first-seen files from being delivered at all (false positives). Quarantine holds the file while the sandbox performs dynamic analysis; if found benign, it is released; if malicious, it is blocked and the hash is added to Cloud Effect for subsequent downloads.

Standard analysis identifies malware after the sandbox detonates the file. Patient Zero protection uses AI Instant Verdict to analyze file behavior indicators in real time before detonation completes, catching fast-moving threats and protecting the first user who downloads the file before the full analysis is done.

Sandbox Malware detections indicate an active download attempt of a novel threat — these require immediate SOC investigation because the user may have been targeted by a new malware campaign. General security events (URL blocks, SSL bypasses) are informational. High-severity targeted events like malware downloads warrant immediate notification to reduce dwell time.

TASK 7.1

Task 7.1: Configure Sandbox Policies

Create a catch-all Sandbox policy rule to quarantine first-seen files and block subsequent downloads of known-malicious content.

  1. In the ZIA Admin Portal, go to Policy > Sandbox.
  2. Click Add Sandbox Rule.
  3. Configure the rule:
    • Rule Name: Catch-all
    • Rule Status: Enabled
    • CRITERIA:
      • File Types: Select all groups — Archive, Executable, Microsoft Office, Mobile, Other Documents, Web Content
      • Sandbox Categories: Select All
      • Protocol: Select All
    • ACTION:
      • First-Time Action: Quarantine
      • Action for Subsequent Downloads: Block
  4. Click Save and Activate.

Task 7.2: Real-Time Alerts

Configure alert definitions and subscriptions so sandbox malware detections trigger email notifications.

  1. In the ZIA Admin Portal, navigate to Administration > Alerts.
  2. Under the Define Alerts tab, click Add Alert Definition.
  3. In the Alert Name field, select "Sandbox Malware". Leave other fields at their default values and click Save.
  4. Click the Publish Alerts tab, then click Add Alert Subscription.
  5. Enter one or more email addresses in the Email field.
  6. Under Security Alerts, enable the Critical selector.
  7. Click Save.

TASK 7.3

Task 7.3: View Sandbox Activity Report

Use the SDC tenant to explore the Sandbox Activity Report and Web Insights sandbox logs.

SDC Lab — Read-Only: For this task you will switch to the Solutions Demo Center (SDC) tenant at https://sdc.zslogin.net. Log in with your Admin Username and Session Password provided in your lab resources. Use the IT Username and Session Password for Zscaler Client Connector.
  1. In the ZIA Admin Portal (SDC tenant), go to Analytics > Sandbox Activity Report.
  2. Explore the graphs — note how many quarantined files were assessed as malicious.
  3. From the dropdown in the upper left, select Sandbox Files found Malicious.
  4. Review the list of malicious files, including their File Type and Threat Name.
    Tip: Clicking a file's Threat Name opens the Zscaler Threat Library with more details about the identified threat.
  5. In the table, click a unique MD5 value for a malicious file and select View Sandbox Detail Report. Review forensic details: registry changes, network connections, MITRE ATT&CK techniques.
  6. Go to Analytics > Web Insights and select the Logs tab.
  7. Click Add Filter, select Sandbox, and select all Sandbox options (Adware, Anonymizer, Benign, Malware, Offensive Security Tools, Ransomware).
  8. Click Done, then Apply Filters.
  9. Change the Timeframe as needed and click the column icon to add/remove columns from the log table.
  10. Scroll through the log details.
