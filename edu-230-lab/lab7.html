💭 THINK FIRST · LAB 7

Sandbox is the last line of defence for unknown files — it trades a small quarantine delay for certainty about first-seen malware before it executes on the endpoint.

  • Why is "Quarantine on first-seen" safer than "Block" for the sandbox First-Time Action?
  • What is Patient Zero protection and how does AI Instant Verdict improve on standard sandbox analysis?
  • Why configure real-time alerts specifically for Sandbox Malware rather than all security events?
Q1 — MODEL ANSWER
A truly novel malicious file has no prior reputation record — blocking would prevent benign first-seen files from being delivered (false positives). Quarantine holds the file while sandbox performs dynamic analysis; if benign, it is released; if malicious, it is blocked and the hash is added to Cloud Effect for subsequent downloads.
Q2 — MODEL ANSWER
Standard analysis identifies malware after the sandbox detonates the file. Patient Zero / AI Instant Verdict uses behavioral indicators in real time before detonation completes, catching fast-moving threats and protecting the first user who downloads the file before full analysis finishes.
Q3 — MODEL ANSWER
Sandbox Malware detections indicate an active download attempt of a novel threat — these require immediate SOC investigation. General security events are informational. High-severity targeted events like malware downloads warrant immediate notification to reduce dwell time.

Task 7.1: Configure Sandbox Policies

Create a catch-all Sandbox rule to quarantine first-seen files and block subsequent downloads of known-malicious content.

  1. In the ZIA Admin Portal, go to Policy > Sandbox.
  2. Click Add Sandbox Rule.
  3. Configure:
    • Rule Name: Catch-all · Status: Enabled
    • File Types: select all groups — Archive, Executable, Microsoft Office, Mobile, Other Documents, Web Content
    • Sandbox Categories: Select All
    • Protocol: Select All
    • First-Time Action: Quarantine
    • Action for Subsequent Downloads: Block
  4. Click Save and Activate.

Task 7.2: Real-Time Alerts

1 Create Alert Definition

  1. In the ZIA Admin Portal, navigate to Administration > Alerts.
  2. Under the Define Alerts tab, click Add Alert Definition.
  3. In the Alert Name field, select "Sandbox Malware". Leave other fields at default values. Click Save.

2 Create Alert Subscription

  1. Click the Publish Alerts tab → Add Alert Subscription.
  2. Enter one or more email addresses in the Email field.
  3. Under Security Alerts, enable the Critical selector.
  4. Click Save.

Task 7.3: View Sandbox Activity Report

SDC Lab — Read-Only: For this task, switch to the Solutions Demo Center (SDC) tenant at https://sdc.zslogin.net. Log in with your Admin Username and Session Password provided in your lab resources.

1 Explore the Sandbox Activity Report

  1. In the ZIA Admin Portal (SDC tenant), go to Analytics > Sandbox Activity Report.
  2. Explore the graphs — note how many quarantined files were assessed as malicious.
  3. From the dropdown in the upper left, select Sandbox Files found Malicious.
  4. Click a unique MD5 value → View Sandbox Detail Report. Review forensic details: MITRE ATT&CK techniques, networking, registry changes.

2 Explore Web Insights Sandbox Logs

  1. Go to Analytics > Web Insights > Logs tab.
  2. Click Add Filter → select Sandbox → select all options (Adware, Anonymizer, Benign, Malware, Offensive Security Tools, Ransomware).
  3. Click Done → Apply Filters.
  4. Change the Timeframe and click the column icon to customise the log table view.

MENTAL NOTE
