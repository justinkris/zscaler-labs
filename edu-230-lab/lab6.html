Lab 6: ZPA Traffic Inspection via ZIA

LAB 6

Scenario: ZPA traffic can be routed through ZIA for full inspection of private application access. This lab provisions an App Connector, configures a ZPA Application Segment for a corporate intranet server, creates an Access Policy, and enables the ZIA Inspected ZPA Apps forwarding rule.

⏱ 50 min 📋 5 tasks ● ZPA
🤔 Think First — Before You Click 3 questions to prime your thinking ▾

ZPA secures access (authentication and authorization) but does not inspect application-layer content. An authenticated user could still transfer malware to a private server, exfiltrate data, or trigger policy violations within the private app. Routing ZPA through ZIA adds content inspection, DLP, and threat protection to private application access.

The provisioning key is a one-time credential used to authenticate and register the App Connector with the ZPA cloud. The ZPA connector software reads this file by exact filename from /opt/zscaler/var/ — if the name is wrong (e.g., provision_key.txt), the service cannot locate the key and will fail to enroll with the ZPA infrastructure.

'Closer to User' selects the App Connector with the lowest latency to the client, minimizing the ZPA tunnel hop for the user. 'Closer to Application' selects based on proximity to the backend server, useful when the application is geographically centralized and consistent server-side latency matters more than variable user-side latency.

TASK 6.1

Task 6.1: Provision an App Connector

Retrieve the App Connector provisioning key from ZPA and save it to the Windows server for transfer to the connector VM.

  1. On the VM labelled Corp: Win Svr 2016, log in as PATRAINING\Administrator with password Admin-123!.
  2. Open Chrome and log in to the ZPA Admin Portal via ZIdentity.
  3. Navigate to Configuration & Control > Private Infrastructure > App Connectors > App Connector Provisioning Keys tab.
  4. Click the Copy icon next to the OnPrem key.
  5. Open Notepad, paste the key, and save as provision_key on the Desktop.
    Caution: The filename is case-sensitive and must be exactly provision_key — NOT provision_key.txt. Save with quotes around the name in the Save dialog to prevent the .txt extension.

TASK 6.2

Task 6.2: Activate the App Connector

Transfer the provisioning key to the App Connector VM and start the connector service.

  1. On the VM labelled Corp: ZPA Connector, log in with default credentials: admin / zscaler.
  2. Start the SSH Daemon: sudo systemctl start sshd (enter password zscaler when prompted).
  3. Verify SSH is active: sudo systemctl status sshd — confirm status is active (running).
  4. On Corp: Win Svr 2016, start WinSCP from the desktop or taskbar.
  5. Load the saved session ZPA Connector (IP: 10.0.0.2, Port 22) and click Login.
  6. Log in with password zscaler, accept the certificate if prompted.
  7. In the left panel (local Windows server), navigate to the Desktop folder and select provision_key.
  8. Right-click > Upload. Set the remote path to /home/admin/ and click OK.
  9. Verify the file is uploaded, then close WinSCP.
  10. Back on Corp: ZPA Connector, stop the SSH Daemon: sudo systemctl stop sshd.
    Note: For security reasons, it is recommended to not leave the SSH Daemon running.
  11. Activate the App Connector with the provisioning key:
    1. pwd — confirm you are in /home/admin
    2. ls — confirm provision_key is present
    3. sudo systemctl stop zpa-connector
    4. sudo cp provision_key /opt/zscaler/var/
      Caution: The filename is case-sensitive: provision_key, NOT provision_key.txt.
    5. sudo ls /opt/zscaler/var/ — verify the file is there
    6. sudo systemctl start zpa-connector
    7. sudo systemctl enable zpa-connector
  12. After ~10 seconds, check the connector status: sudo systemctl status zpa-connector — verify active (running) and a connection to the ZPA infrastructure.
  13. On Corp: Win Svr 2016, in the ZPA Admin Portal, navigate to Configuration & Control > Private Infrastructure > App Connectors — confirm the connector is listed as Connected.
  14. Check the Upgrade Status. If Scheduled, click Update Now, confirm the checkbox, and click Update Now.

TASK 6.3

Task 6.3: Add an Intranet Application

Create a ZPA Application Segment for the corporate intranet server so it can be accessed through the App Connector.

  1. In the ZPA Admin Portal, go to Resource Management > Application Management > Application Segments.
  2. Click + Add Application Segment.
  3. At the Define Applications step:
    • Name: Intranet
    • Status: Enabled
    • Inspect Traffic with ZIA: Enabled
    • Applications (domain/IP): intranet.patraining.safemarch.com
      Caution: There is NO pod number in the domain.
    • CLIENT CONNECTOR ACCESS — Default Port Ranges: Web Server (adds TCP 80, 443, 8080)
    • TCP Keepalive: Disabled
    • ADDITIONAL CONFIGURATION: Double Encryption — Disabled; Bypass — Use Client Forwarding Policy; ICMP Access — Disabled
    • COMMON CONFIGURATION: Health Reporting — Continuous; Client Connector can receive CNAME — Enabled; App Connector Selection Method — Closer to User
  4. Click Next to the Segment Group step. Click Add Segment Group, name it CorpApps, Status Enabled, click Next.
  5. At the Server Groups step, click Add Server Group:
    • Name: CorpServers
    • Status: Enabled
    • Dynamic Server Discovery: On
    • App Connector Groups: OnPrem
  6. Click Next, then at the Review step click Save.
  7. Click Edit Policy to proceed to Access Policy configuration.

TASK 6.4

Task 6.4: Add an Access Policy for the Intranet Application

Create a ZPA Access Policy rule that allows users to connect to the Intranet application segment.

  1. On the Access Policy page, click + Add.
  2. Configure the policy rule:
    • Name: Allow Intranet
    • Rule Action: Allow Access
    • App Connector Selection Method: All App Connector groups for the application
    • Message to User: leave blank
  3. Click Add Criteria, select Applications.
  4. In the Application Segments field, select Intranet.
  5. Do not add any additional criteria.
  6. Click Save.
  7. On the Corp: Client PC, open a browser and navigate to http://intranet.patraining.safemarch.com — confirm the Intranet page loads.
  8. Also test with HTTPS and confirm it loads. Close the window.

TASK 6.5

Task 6.5: Enable ZIA Inspected ZPA Apps

Enable the predefined ZIA Inspected ZPA Apps forwarding rule so that ZPA traffic is routed through ZIA for inspection.

  1. In the ZIA Admin Portal, navigate to Policy > Forwarding Control.
  2. Find the row ZIA Inspected ZPA App and click the pencil (edit) icon.
  3. Change the Rule Status to Enabled.
  4. Click Save then Activate the changes.
  5. Navigate to Policy > SSL Inspection and click on the Enable SSL Rule.
  6. Scroll down to the Action section. For Untrusted Server Certificates, select Allow.
  7. Click Save then Activate.
  8. Open a Chrome window and navigate to http://intranet.patraining.safemarch.com — confirm the page loads.
  9. From ZIdentity, open the ZPA Admin Portal.
  10. Navigate to Analytics > Diagnostics.
  11. Click Add Filters and select ZIA Inspection: True, then click Apply.
  12. Review the list of connections that are ZIA Inspected — expand an entry to see the ZIA INSPECTION: Enabled field.
