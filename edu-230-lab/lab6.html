Lab 6: ZPA Traffic Inspection via ZIA

LAB 6

Scenario: ZPA secures access to private applications, but does not inspect application-layer content. By routing ZPA traffic through ZIA, you add full content inspection, DLP, and threat protection to private app access. This lab provisions an App Connector, creates an Application Segment, and enables the ZIA Inspected ZPA Apps forwarding rule.

Time: ~50 min
Phase: ZPA
Tasks: 5
💭 THINK FIRST · LAB 6

ZPA provides access control; ZIA provides content inspection. Together they deliver zero trust for both public internet and private application traffic through a single policy framework.

  • Why would you route ZPA private application traffic through ZIA for inspection?
  • What is the role of the App Connector Provisioning Key, and why is the filename case-sensitive?
  • What does "Closer to User" App Connector Selection Method mean vs "Closer to Application"?
Q1 — MODEL ANSWER
ZPA authenticates and authorises access but does not inspect application-layer content. An authenticated user could still transfer malware to a private server or exfiltrate data. Routing ZPA through ZIA adds content inspection, DLP, and threat protection to private app access.
Q2 — MODEL ANSWER
The provisioning key is a one-time credential used to authenticate and register the App Connector with the ZPA cloud. The connector software reads this file by exact filename from /opt/zscaler/var/ — a wrong name like provision_key.txt means the service cannot locate the key and fails to enroll.
Q3 — MODEL ANSWER
"Closer to User" selects the connector with lowest latency to the client, minimising the ZPA tunnel hop. "Closer to Application" selects based on server proximity — useful when the application is geographically centralised and server-side consistency matters more than variable user-side latency.

Task 6.1: Provision an App Connector

Retrieve the App Connector provisioning key from ZPA and save it to the Windows server for transfer to the connector VM.

  1. On Corp: Win Svr 2016, log in as PATRAINING\Administrator / Admin-123!.
  2. Open Chrome and log in to the ZPA Admin Portal via ZIdentity.
  3. Navigate to Configuration & Control > Private Infrastructure > App Connectors > App Connector Provisioning Keys tab.
  4. Click the Copy icon next to the OnPrem provisioning key.
  5. Open Notepad, paste the key, and save to the Desktop as provision_key (use quotes around the name in the Save dialog to prevent the .txt extension).
    Caution: The filename is case-sensitive — must be exactly provision_key, NOT provision_key.txt.

Task 6.2: Activate the App Connector

1 Enable SSH on the Connector VM

  1. On Corp: ZPA Connector, log in as admin / zscaler.
  2. sudo systemctl start sshd (password: zscaler)
  3. sudo systemctl status sshd — confirm active (running).

2 Transfer the provisioning key via WinSCP

  1. On Corp: Win Svr 2016, launch WinSCP.
  2. Load the ZPA Connector saved session (IP: 10.0.0.2, Port: 22) and click Login. Accept the certificate.
  3. In the left panel (local), navigate to the Desktop and select provision_key.
  4. Right-click > Upload → set remote path to /home/admin/ → click OK.
  5. Verify the file appears in the right panel, then close WinSCP.

3 Activate the connector

  1. On Corp: ZPA Connector, stop SSH: sudo systemctl stop sshd
    Note: For security, do not leave SSH running after the file transfer.
  2. pwd — confirm you are in /home/admin
  3. ls — confirm provision_key is present
  4. sudo systemctl stop zpa-connector
  5. sudo cp provision_key /opt/zscaler/var/
  6. sudo ls /opt/zscaler/var/ — confirm the file is there
  7. sudo systemctl start zpa-connector
  8. sudo systemctl enable zpa-connector
  9. After ~10 seconds: sudo systemctl status zpa-connector — verify active (running).

4 Confirm in ZPA Admin Portal

  1. On Corp: Win Svr 2016, navigate to Configuration & Control > Private Infrastructure > App Connectors — confirm status is Connected.
  2. If Upgrade Status shows Scheduled, click Update Now, confirm the checkbox, and click Update Now.

Task 6.3: Add an Intranet Application

Create a ZPA Application Segment for the corporate intranet server.

  1. In the ZPA Admin Portal, go to Resource Management > Application Management > Application Segments > + Add Application Segment.
  2. Define Applications step:
    • Name: Intranet · Status: Enabled · Inspect Traffic with ZIA: Enabled
    • Applications domain: intranet.patraining.safemarch.com
      Caution: There is NO pod number in this domain.
    • Default Port Ranges: Web Server (adds TCP 80, 443, 8080) · TCP Keepalive: Disabled
    • Double Encryption: Disabled · Bypass: Use Client Forwarding Policy · ICMP Access: Disabled
    • Health Reporting: Continuous · Client Connector can receive CNAME: Enabled · App Connector Selection Method: Closer to User
  3. Click Next. At Segment Group, click Add Segment Group → name it CorpApps → Status Enabled → Next.
  4. At Server Groups, click Add Server Group:
    • Name: CorpServers · Status: Enabled · Dynamic Server Discovery: On
    • App Connector Groups: OnPrem
  5. Click Next → at Review, click Save → click Edit Policy.

Task 6.4: Add an Access Policy for the Intranet Application

  1. On the Access Policy page, click + Add.
  2. Configure:
    • Name: Allow Intranet · Rule Action: Allow Access
    • App Connector Selection Method: All App Connector groups for the application
    • Message to User: leave blank
  3. Click Add Criteria → select Applications.
  4. In the Application Segments field, select Intranet. Do not add additional criteria.
  5. Click Save.
  6. On Corp: Client PC, navigate to http://intranet.patraining.safemarch.com — confirm the page loads. Also test HTTPS.

Task 6.5: Enable ZIA Inspected ZPA Apps

1 Enable the Forwarding Rule

  1. In the ZIA Admin Portal, go to Policy > Forwarding Control.
  2. Find the row ZIA Inspected ZPA App and click the pencil icon.
  3. Set Rule Status to Enabled. Click Save then Activate.

2 Update the SSL Inspection Rule

  1. Go to Policy > SSL Inspection and click on the Enable SSL Rule.
  2. Under the Action section, for Untrusted Server Certificates, select Allow.
  3. Click Save then Activate.

3 Verify ZIA Inspection in ZPA Logs

  1. On Corp: Client PC, navigate to http://intranet.patraining.safemarch.com — confirm it loads.
  2. Open the ZPA Admin Portal > Analytics > Diagnostics.
  3. Click Add Filters → select ZIA Inspection: True → click Apply.
  4. Expand a connection entry and verify the ZIA INSPECTION: Enabled field is visible.
