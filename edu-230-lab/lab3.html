Lab 3: DNS Security

LAB 3

Scenario: DNS is a critical attack vector. In this lab you configure DNS control policies to block malicious and unwanted domains at the DNS layer, preventing resolution before any TCP connection is established.

⏱ 20 min 📋 3 tasks ● ZIA
🤔 Think First — Before You Click 3 questions to prime your thinking ▾

DNS blocking prevents resolution before any TCP/IP connection is established — the client never sends a packet to the malicious server. URL filtering operates later (at the HTTP/HTTPS layer) after a connection is already formed. DNS blocking also catches non-web protocols that use the same malicious domain.

It tells Zscaler Client Connector to forward ALL DNS queries (for all domains) through ZIA's DNS resolver. Without this, only specific domains you list would be subject to ZIA's DNS policies — the wildcard ensures universal DNS inspection coverage.

Attackers can use DNS-over-HTTPS (DoH) or hardcoded IP addresses to bypass DNS-layer controls. Additionally, fast-flux domains change IPs rapidly. These gaps are addressed by additional layers: Cloud Firewall to block DoH endpoints, and URL/IP reputation policies.

TASK 3.1

Task 3.1: Configure App Profile

Enable domain inclusion in the Zscaler Client Connector App Profile so that DNS queries are forwarded through ZIA.

  1. In ZIdentity > Zscaler Client Connector, go to App Profiles > Windows.
  2. Edit the HandsOnLab policy and navigate to the DNS tab.
  3. Under Domain Inclusion, type * in the field and press Enter to create the wildcard entry.
  4. Click Save.
  5. On the endpoint, open Zscaler Client Connector and click More > Update Policy.

TASK 3.2

Task 3.2: Configure DNS Control Policy

Create a DNS filtering rule that blocks gambling-category domains at the DNS layer.

  1. Pre-test: Open a command prompt and run:
    nslookup www.gambling.com 8.8.8.8
    Confirm it resolves (no policy yet).
  2. In the ZIA Admin Portal, go to Policy > Firewall > DNS Control.
  3. Click + Add DNS Filtering Rule:
    • Rule Order: 1
    • Name: Block Gambling
    • DNS Application tab — Request Categories and Response Categories: Gambling
    • Action: Block
  4. Click Save then Activate.
  5. Post-test: Run ipconfig /flushdns then nslookup www.gambling.com 8.8.8.8 — the query should now time out.
  6. Verify a legitimate domain still resolves: nslookup www.sdxcentral.com 8.8.8.8

TASK 3.3

Task 3.3: DNS Insight Logs

Review the DNS Insights logs to confirm blocked gambling requests are being logged.

  1. In the ZIA Admin Portal, go to Analytics > Insights > DNS Insights > Logs.
  2. Set Timeframe to Current Day.
  3. Add a filter: Rule Name: Block Gambling.
  4. Click Apply Filters.
  5. Review the log entries showing blocked gambling domain requests (e.g., www.888.com).
