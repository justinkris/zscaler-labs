Lab 2: Configuring SSL Inspection Policies

LAB 2

Scenario: SSL/TLS inspection is essential for visibility into encrypted traffic. In this lab you configure a Forwarding Profile, enable SSL inspection globally, create exemptions for pinned-certificate applications, and verify the Zscaler CA is trusted on the endpoint.

⏱ 45 min 📋 5 tasks ● ZIA
🤔 Think First — Before You Click 3 questions to prime your thinking ▾

Z-Tunnel 2.0 with Packet Filter Based driver allows ZIA to enforce Cloud Firewall policies on all traffic from the endpoint, not just HTTP/HTTPS. Without this combination, non-web protocols and ports would bypass ZIA inspection entirely.

Certificate pinning is a security mechanism where an application validates the server certificate against a hard-coded expected value. When Zscaler intercepts traffic and presents its intermediate CA certificate instead, the pinned check fails and the application rejects the connection — triggering the 'failed to connect' error you see in Signal and Adobe.

You would review Web Insights logs for 'Dropped due to failed client SSL handshake' entries, pilot SSL inspection on a small user group first, consult vendor documentation for known pinned apps, and maintain an SSL bypass list that grows as you discover pinned applications.

TASK 2.1

Task 2.1: Configure Forwarding Options

Create a Forwarding Profile and App Profile in Zscaler Client Connector so traffic is sent through ZIA via Z-Tunnel 2.0.

  1. Log in to ZIdentity and open Zscaler Client Connector.
  2. Go to Administration > Forwarding Profile > Add.
  3. Name the profile HandsOnLab and configure:
    • Windows Driver: Packet Filter Based
    • ZIA On-Trusted Network: Tunnel
    • Tunnel version: Z-Tunnel 2.0
    • VPN / Off-Trusted Network: Same as On-Trusted
  4. Save the profile.
  5. Go to App Profiles > Windows > Add Windows Policy:
    • Name: HandsOnLab
    • Rule Order: 1
    • Forwarding Profile: HandsOnLab
    • Install Zscaler SSL Cert: On
    • User Groups: All
  6. Save the App Profile.
  7. Log in to Zscaler Client Connector on the endpoint, verify Service Status ON for both Private Access and Internet Security.
  8. Click More > Update Policy to pull the new profile.

TASK 2.2

Task 2.2: Enable SSL Inspection for All Destinations

Enable global SSL inspection so Zscaler can decrypt and inspect all HTTPS traffic.

  1. First verify no SSL inspection is active: visit x.com and check the certificate — it should show Let's Encrypt as the issuer.
  2. In the ZIA Admin Portal, go to Policy > Web > ACCESS CONTROL > SSL Inspection.
  3. Click + Add SSL Inspection Rule:
    • Rule Order: 1
    • Name: Enable SSL
    • Action: Inspect
  4. Click Save then Activate.
  5. Revisit x.com — the certificate issuer should now show Zscaler Intermediate Root CA.

TASK 2.3

Task 2.3: Enable SSL Exemption

Some categories (like healthcare) should not be decrypted for privacy compliance. Create a bypass rule.

  1. In the ZIA Admin Portal, go to Policy > SSL Inspection.
  2. Click + Add SSL Inspection Rule:
    • Rule Order: 1 (above Enable SSL)
    • Name: SSL Bypass for Health Care
    • URL Categories: Health
    • Action: Do Not Inspect
  3. Click Save then Activate.
  4. Visit health.com — the certificate should now show R3 / Let's Encrypt (not Zscaler).

TASK 2.4

Task 2.4: Verify Zscaler CA Installation

Confirm the Zscaler Root Certificate is installed in the Windows trust store.

  1. Press Win + R, type certmgr, and press Enter.
  2. Navigate to Trusted Root Certification Authorities > Certificates.
  3. Confirm the Zscaler Root CA certificate is present.
  4. Alternatively, in Chrome go to chrome://certificate-manager/ > Local certificates > View imported from Windows — confirm you see the Zscaler Root CA listed.

TASK 2.5

Task 2.5: Certificate Pinning Error

Some applications use certificate pinning and fail when Zscaler intercepts their traffic. Learn to identify and resolve these issues.

Signal Messenger

  1. Download Signal from signal.org and attempt to install — you will see: "Something went wrong! Failed to connect to server".
  2. In the ZIA Admin Portal, add a new SSL Inspection Rule:
    • Rule Order: 1
    • Name: Enable Signal
    • Cloud Applications: Signal (under Instant Messaging)
    • Action: Do Not Inspect
  3. Save and Activate.
  4. End the Signal task in Task Manager, then re-install — it should now succeed.
  5. Go to Analytics > Web Insights > Logs, set Timeframe to Current Day, add filters for SSL Inspected and Bypassed Transaction. Find the "Dropped due to failed client SSL handshake" entries for signal.org.

Adobe Acrobat

  1. Download Adobe Reader from the Adobe website — you will see: "Application initialization error".
  2. Create a custom URL category "Adobe URL List" containing the Adobe hosted domains listed on the Adobe Product Support page.
  3. Add an SSL Inspection Rule:
    • Name: Enable Adobe
    • URL Category: Adobe URL List
    • Action: Do Not Inspect
  4. End the Adobe Download Manager task, then re-install Adobe Reader — it should now succeed.
