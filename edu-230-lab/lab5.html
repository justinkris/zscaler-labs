💭 THINK FIRST · LAB 5

Browser Isolation is the safest middle ground between blocking access and allowing unrestricted browsing — users see content but the endpoint never executes remote code.

  • Why might Browser Isolation be preferred over simply blocking a URL category entirely?
  • What is the difference between URL Filtering-based isolation and Cloud App Control-based isolation?
  • How does Smart Browser Isolation's AI/ML approach improve on static policy-based isolation?
Q1 — MODEL ANSWER
Isolation allows users to access content (maintaining productivity) while preventing any data from reaching the local endpoint. A hard block creates frustration and shadow IT workarounds. Isolation is a glass wall: visible but contained.
Q2 — MODEL ANSWER
URL Filtering isolation triggers on URL category matches (broad — any Social Networking site). Cloud App Control isolation triggers on specific identified applications (precise — only Twitter/X or Reddit). Using both ensures you catch category-matched sites AND specifically identified apps even if miscategorized.
Q3 — MODEL ANSWER
Static policy requires manual identification of risky URLs. AI/ML evaluates real-time signals (domain age, reputation, content analysis) to automatically isolate unknown or suspicious sites without requiring a policy update — responding to newly registered malicious domains instantly.

Task 5.1: Build Isolation Profile

Create an isolation profile that renders content in the cloud browser while blocking data exfiltration paths.

  1. In the ZIA Admin Portal, go to Administration > Browser Isolation > Add Profile.
  2. Name: Render Only · Description: Render docs in Isolation, no download / upload / copy / paste.
  3. Under Company Settings, set Root Certificates File (.pem) to Zscaler Root Certificate.
  4. Under Regions, select Any. Leave Security settings at default.
  5. Click Save.

Task 5.2: Implement Isolation Policy

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud App Control > URL Filtering Policy > Add URL Filtering Rule.
  2. Configure:
    • Name: Isolate Test
    • URL Categories: Social Networking and Social Networking Adult
    • Protocols: HTTP, HTTPS
    • User Agent: Chrome, Firefox, Microsoft Edge, Safari
    • Action: Isolate · Isolation Profile: Render Only
  3. Click Save then Activate.

Task 5.3: Add URL/Cloud App Isolate Control Policies

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud Application Control > Cloud App Control Policy > Add > Social Networking.
  2. Configure:
    • Rule Name: Social_Networking_2
    • Cloud Applications: Twitter (X) and Reddit
    • User Agent: Chrome, Firefox, Edge, Safari
    • Application Access: Isolate · Isolation Profile: Render Only
  3. Click Save then Activate.
  4. Test: navigate to reddit.com or twitter.com — the page should load inside Browser Isolation with the banner: "Heads up, you've been redirected to Browser Isolation!"

Task 5.4: Smart Browser Isolation

1 Test Without Smart Isolation

  1. Open an incognito window and navigate to https://work-nutjobber.store — it should load normally (no isolation yet for unknown sites).

2 Enable Smart Isolation

  1. In the ZIA Admin Portal, go to Policy > Secure Browsing > Smart Isolate tab.
  2. Enable AI/ML based Smart Browser Isolation.
  3. Click Save then Activate.

3 Verify Smart Isolation

  1. Open a new incognito window and navigate to https://work-nutjobber.store again — it should now be automatically redirected to Browser Isolation.

MENTAL NOTE
Isolation is a spectrum: URL category isolation (broadest), Cloud App isolation (per-app), Smart Isolation (AI-driven for unknown sites). Stack all three for comprehensive coverage. The Render Only profile prevents data exfiltration without blocking access.
