Lab 5: Browser Isolation

LAB 5

Scenario: Zscaler Browser Isolation renders web content in a remote cloud browser, protecting the endpoint from threats while allowing users to view content. This lab configures isolation profiles, URL-based isolation policies, Cloud App isolation, and Smart Browser Isolation.

⏱ 30 min 📋 4 tasks ● ZIA
🤔 Think First — Before You Click 3 questions to prime your thinking ▾

Isolation allows users to access content (maintaining productivity) while preventing any data from reaching the local endpoint — no downloads, no clipboard access, no DOM execution. A hard block creates frustration and shadow IT workarounds. Isolation is a middle ground: visible but contained.

URL Filtering isolation triggers on URL category matches (broad — any Social Networking site). Cloud App Control isolation triggers on specific identified applications (precise — only Twitter/X or Reddit). Using both layers ensures you catch category-matched sites AND specifically identified applications even if they are miscategorized.

Static policy requires an admin to manually identify and categorize risky URLs — it cannot respond to newly registered malicious domains. AI/ML-based Smart Isolation evaluates real-time signals (domain age, reputation, content analysis) to automatically isolate unknown or suspicious sites without requiring a policy update.

TASK 5.1

Task 5.1: Build Isolation Profile

Create an isolation profile that renders content in the cloud browser but prevents data exfiltration.

  1. In the ZIA Admin Portal, go to Administration > Browser Isolation > Add Profile.
  2. Configure the profile:
    • Name: Render Only
    • Description: Render docs in Isolation, but not allow download, upload, copy, or paste
  3. Under Company Settings, set Root Certificates File (.pem) to Zscaler Root Certificate.
  4. Under Security, leave defaults.
  5. Under Regions, select Any.
  6. Click Save.

TASK 5.2

Task 5.2: Implement Isolation Policy

Apply the isolation profile to social networking URL categories.

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud App Control > URL Filtering Policy.
  2. Click + Add URL Filtering Rule:
    • Name: Isolate Test
    • URL Categories: Social Networking and Social Networking Adult
    • Protocols: HTTP, HTTPS
    • User Agent: Chrome, Firefox, Microsoft Edge, Safari
    • Action: Isolate
    • Isolation Profile: Render Only
  3. Click Save then Activate.

TASK 5.3

Task 5.3: Add URL/Cloud App Isolate Control Policies

Apply isolation at the Cloud App Control layer for specific social networking apps.

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud Application Control > Cloud App Control Policy.
  2. Click + Add > Social Networking:
    • Rule Name: Social_Networking_2
    • Cloud Applications: Twitter (X) and Reddit
    • User Agent: Chrome, Firefox, Microsoft Internet Explorer, Microsoft Edge, Safari
    • Action — Application Access: Isolate
    • Isolation Profile: Render Only
  3. Click Save then Activate.
  4. Test: navigate to reddit.com or twitter.com — the page should load inside the Browser Isolation environment with the banner: "Heads up, you've been redirected to Browser Isolation!"

TASK 5.4

Task 5.4: Smart Browser Isolation

Enable AI/ML-based Smart Browser Isolation to automatically isolate suspicious unknown sites.

  1. Open an incognito window and navigate to https://work-nutjobber.store — it loads normally (no isolation yet).
  2. In the ZIA Admin Portal, go to Policy > Secure Browsing > Smart Isolate tab.
  3. Enable AI/ML based Smart Browser Isolation.
  4. Click Save then Activate.
  5. Re-open https://work-nutjobber.store in a new incognito window — it should now be automatically redirected to Browser Isolation.
