Lab 4: URL Filtering and Cloud App Control

LAB 4

Scenario: URL filtering and cloud application control give you granular policy enforcement over web categories and specific SaaS applications. This lab configures blocking rules, allows specific cloud app actions, and confirms your improved security posture.

⏱ 35 min 📋 6 tasks ● ZIA
🤔 Think First — Before You Click 3 questions to prime your thinking ▾

Cloud App Control policies are evaluated before URL Filtering in the ZIA policy processing order. They operate at a more specific application layer — identifying the exact cloud application rather than just the URL category — so a specific Cloud App Allow will override a broader URL category Block.

Users may use personal hotspots, VPNs, or other bypass methods to reach blocked content, removing them from ZIA visibility entirely. A nuanced approach — blocking uploads/sharing while allowing reading — maintains both productivity and security visibility.

securitypreview.zscaler.com sends controlled test traffic representing known threat patterns (malware download attempts, phishing page loads, XSS probes, botnet callbacks). If ZIA intercepts and blocks these patterns per the configured policies, the test passes. It is a functional validation, not a theoretical configuration check.

TASK 4.1

Task 4.1: View Threat Protection Configurations & Risk Reports

Review the current threat protection settings and understand the risk posture before making changes.

  1. In the ZIA Admin Portal, go to Policy > Malware Protection — review settings and compare to the Recommended Policy.
  2. Go to Policy > Advanced Threat Protection — review available settings and note the Page Risk Score Index.
  3. Go to Analytics > Configuration Risk Report.
  4. Click through each risk category: Web-Based, File-Based, Network-Based, Uninspected Encrypted.
  5. Identify the highest-risk category and review the recommended remediation actions.

TASK 4.2

Task 4.2: Configure URL Filtering Controls

Create URL filtering rules to block news media and video streaming sites.

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud App Control > URL Filtering Policy.
  2. Click + Add URL Filtering Rule:
    • Name: Block Website
    • Rule Order: 1
    • URL Categories: News and Media
    • All request methods/protocols
    • Action: Block
  3. Save the rule.
  4. Add another URL Filtering Rule:
    • Name: Block YouTube
    • Rule Order: 1
    • URL Categories: Video Streaming
    • Action: Block
  5. Save and Activate the changes.

Task 4.3: Test End User Experience with URL Filtering

Verify the URL filtering rules are enforced from the endpoint.

TASK 4.4

Task 4.4: Configure Cloud App Control

Use Cloud App Control to allow specific actions on YouTube and ChatGPT while blocking others.

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud App Control > Cloud App Control Policy.
  2. Click + Add > Streaming Media:
    • Name: Allow YouTube
    • Cloud Apps: YouTube
    • Action: Viewing/Listening — Allow; Uploading — Block
  3. Save the rule.
  4. Click + Add > AI & ML:
    • Name: ChatGPT Permissions
    • Cloud Apps: ChatGPT
    • Actions: App Access — Allow; Chatting — Allow; Downloading / Deleting / Inviting / Sharing / Uploading — Block
  5. Save and Activate the changes.

TASK 4.5

Task 4.5: Test End User Experience with Cloud App Control

Verify Cloud App Control policies are enforced (Cloud App rules take priority over URL filtering rules).

  1. Navigate to youtube.com — it should now be accessible (Cloud App Allow overrides URL block).
  2. Navigate to chatgpt.com and type a message such as "what is cybersecurity" — the response should display (Chatting: Allow).
  3. Try to download an image or file from ChatGPT — it should be blocked with an upload/download error.

Task 4.6: Check Your Security Posture

Re-run the security posture tests from Lab 1 to confirm improvement.

  1. Go to securitypreview.zscaler.com and click Test your cyber risk posture again.
  2. Verify these tests now pass:

    • Prevent XSS
    • Prevent common virus from non-encrypted site
