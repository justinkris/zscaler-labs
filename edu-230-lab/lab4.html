Lab 4: URL Filtering and Cloud App Control

LAB 4

Scenario: URL filtering and cloud application control give you granular enforcement over web categories and specific SaaS applications. This lab configures category blocks, cloud app action overrides, and validates the security posture improvement from the earlier labs.

Time: ~35 min
Phase: ZIA
Tasks: 6
💭 THINK FIRST · LAB 4

URL filtering and Cloud App Control are the most visible policies to end users — get the balance right between security and productivity to avoid shadow IT.

  • Why does a Cloud App Control rule for YouTube take priority over a URL Filtering rule that blocks Video Streaming?
  • What is the risk of blocking all News and Media without a Cloud App Control override in a corporate environment?
  • How does securitypreview.zscaler.com actually test your security posture?
Q1 — MODEL ANSWER
Cloud App Control policies are evaluated before URL Filtering in ZIA's policy processing order. They operate at the application layer — identifying the exact cloud app rather than just the URL category — so a specific Cloud App Allow overrides a broader category Block.
Q2 — MODEL ANSWER
Users may switch to personal hotspots, VPNs, or other bypass methods to reach blocked content, removing them from ZIA visibility entirely. Granular controls maintain both productivity and security visibility.
Q3 — MODEL ANSWER
It sends controlled test traffic representing known threat patterns — malware download attempts, phishing page loads, XSS probes, botnet callbacks. If ZIA blocks them per your policies, the test passes. It is a functional validation, not a configuration check.

Task 4.1: View Threat Protection Configurations & Risk Reports

  1. In the ZIA Admin Portal, go to Policy > Malware Protection — review the current settings vs the Recommended Policy guidance.
  2. Go to Policy > Advanced Threat Protection — review available settings and note the Page Risk Score Index.
  3. Go to Analytics > Configuration Risk Report.
  4. Click through each category: Web-Based, File-Based, Network-Based, Uninspected Encrypted.
  5. Identify the highest-risk category and review the recommended remediation actions.

Task 4.2: Configure URL Filtering Controls

  1. In the ZIA Admin Portal, go to Policy > URL & Cloud App Control > URL Filtering Policy.
  2. Click + Add URL Filtering Rule and configure:
    • Name: Block Website · Order: 1
    • URL Categories: News and Media
    • All request methods/protocols · Action: Block
  3. Save the rule.
  4. Add a second rule:
    • Name: Block YouTube · Order: 1
    • URL Categories: Video Streaming
    • Action: Block
  5. Save and Activate.

Task 4.4: Configure Cloud App Control

1 YouTube — Allow Viewing

  1. Go to Policy > URL & Cloud App Control > Cloud App Control Policy > Add > Streaming Media.
  2. Configure: Name Allow YouTube · Cloud Apps YouTube · Viewing/Listening Allow · Uploading Block.
  3. Save.

2 ChatGPT — Granular Actions

  1. Click Add > AI & ML.
  2. Configure: Name ChatGPT Permissions · Cloud Apps ChatGPT.
  3. Actions: App Access Allow · Chatting Allow · Downloading / Deleting / Inviting / Sharing / Uploading Block.
  4. Save and Activate.

Task 4.5: Test End User Experience with Cloud App Control

  1. Navigate to youtube.com — it should now load (Cloud App Allow overrides the URL category Block).
  2. Navigate to chatgpt.com and type a message, e.g. "what is cybersecurity" — the response should display.
  3. Try to download an image or file from ChatGPT — it should be blocked.
Key insight: Cloud App Control rules are evaluated before URL Filtering rules. A specific Cloud App Allow will always override a broader URL category Block.
