💡 THINK FIRST · LAB ARCHITECTURE

The EDU-202 lab uses a Skytap cloud environment with multiple VMs interconnected to simulate a real enterprise network — a corporate LAN, a remote network, and internet-facing segments.

  • Why does the ZPA App Connector sit in the corporate network but requires no inbound firewall rules from the internet?
  • The log streaming connector is separate from the ZPA connector. What would break if they were on the same VM?
  • If the Corp: Client PC can only reach the internet through ZIA, what happens to HVAC browser-access traffic from a remote user who has no ZCC installed?
Q1 — MODEL ANSWER
The App Connector initiates outbound TLS connections to the ZPA cloud — it is a client, not a server. The ZPA cloud broker then stitches the user's ZCC tunnel to the connector's persistent tunnel, so traffic flows without any inbound port exposure on the corporate firewall.
Q2 — MODEL ANSWER
Log streaming sends high-volume syslog data continuously. If the same VM ran ZPA connector services, a log burst could starve the connector's CPU/memory and drop ZPA sessions. Separation lets each service be sized independently.
Q3 — MODEL ANSWER
Browser Access redirects the user's HTTP/HTTPS requests through a Zscaler cloud edge node (not through ZIA's client-based inspection). The CNAME for the HVAC app resolves to a Zscaler BA node, so the remote user's traffic is proxied through ZPA cloud — no ZCC required.

Lab Topology

The EDU-202 lab runs in Skytap and consists of two simulated networks:

  • Corporate LAN — Corp: Client PC (Windows 10), Corp: Windows Server 2019, ZPA App Connector VM (Linux), Log Streaming Connector VM (Linux)
  • Remote Network — Remote: Linux Client PC, Remote: DNS Server (Windows)

The corporate tenant domain is safemarch.com. Your assigned FQDN follows the pattern zsXXXX.safemarch.com. Enter it in the Lab Configuration panel above so all placeholders fill in automatically.

LAB PROGRESS
QUICK REFERENCE ▲
ZIdentity
Zscaler's cloud IdP — authenticates users and authorizes service + admin entitlements for ZIA, ZPA, ZDX, ZCC, and ZWA.
Student FQDN
Your unique tenant domain, e.g. zs9901.safemarch.com. All login IDs are username@FQDN.
ZCC
Zscaler Client Connector — endpoint agent that tunnels traffic through ZTE. Required for ZIA, ZPA private access, and ZDX monitoring.
Z-Tunnel 2.0
Network-layer tunnel intercepting all IP protocols. Required for Cloud Firewall Filtering of non-web traffic.
Browser Access
ZPA feature enabling access to internal web apps without ZCC. Traffic goes through Zscaler cloud via CNAME DNS redirect.
App Connector
ZPA software in the data centre making outbound TLS tunnels to ZPA cloud. No inbound firewall rules needed.
App Segment
ZPA object defining a private resource (domain/IP + ports). Paired with Access Policy to enable ZPA access.
SSL Inspection
Trusted MitM: ZIA decrypts, inspects, re-encrypts using Zscaler CA. Certificate Issuer changes to "Zscaler Intermediate Root CA".
DLP Engine
Defines what sensitive content looks like (dictionaries + threshold). Paired with a DLP Rule that defines the action (Block/Allow).
ZWA
Zscaler Workflow Automation — DLP incident management portal with escalation, notification, and automated workflow templates.
Risk360 / MITRE ATT&CK
Risk360 maps TTPs from MITRE ATT&CK and NIST CSF to your Zscaler configurations, showing coverage and misconfiguration gaps.
ZDX Score
0–100 composite metric: 0=worst, 100=best. Probes measure DNS time, TCP connect, server response, page fetch time continuously.
OneAPI
Unified gateway for all Zscaler REST APIs (ZIA, ZPA, ZDX, ZCC, ZIdentity). OAuth 2.0 via ZIdentity API Clients.
SDC
Solutions Demo Center — shared Zscaler tenants with pre-configured data. Used for Labs 9 (Deception), 11 (ZWA), 12 (Risk360).
Log Streaming
Pushes ZIA/ZPA log records to SIEM in real time via syslog or HTTPS. Requires NSS VM for ZIA, native for ZPA.
