LAB 13

Scenario: Safemarch wants to monitor Gmail performance for all employees. Create a custom ZDX application for Mail_server and configure both a Web probe and a Cloud Path probe to measure performance metrics from the Corp: Client PC.

Time: ~25 minutes
💡 THINK FIRST · ZDX APPLICATION MONITORING

ZDX uses synthetic probes — ZCC runs them in the background on the user's device — to continuously measure application performance. A Web probe measures HTTP response time; a Cloud Path probe traces ICMP hops to isolate where latency originates.

  • What is the difference between a Web probe and a Cloud Path probe, and what specific problem does each one help diagnose?
    Q1 — MODEL ANSWER
    A Web probe sends an HTTP/HTTPS GET request to a specific URL and measures DNS time, TCP connect time, server response time, and total page fetch time — it tells you whether the application is responding and how fast. A Cloud Path probe traces the network hops (ICMP ping) between the user's ZCC and the application server, measuring per-hop latency and packet loss — it tells you where in the network path the latency originates (last-mile ISP, Zscaler node, or data centre).
  • You configure a Web probe with a 60-second timeout and 2 attempts. A user reports slowness. The probe shows Page Fetch Time = 58s. What does this tell you about the problem?
    Q2 — MODEL ANSWER
    A 58-second fetch time is just under the 60s timeout, meaning the application nearly timed out. This is a severe performance problem — the server is responding but extremely slowly. The Cloud Path probe results would help determine if this is a network path issue (high hop latency, packet loss) or a server-side issue (all hops fine but the destination server is slow to respond).
  • ZDX probes run every 5 minutes. A performance issue occurs and resolves within 3 minutes. What is the implication for ZDX's ability to capture this event?
    Q3 — MODEL ANSWER
    ZDX may miss a sub-5-minute performance event entirely if it occurs and resolves between two probe cycles. This is the fundamental limitation of synthetic probe-based monitoring: the resolution of detection is bounded by the probe interval. For faster detection, reduce the probe interval (min 1 minute in ZDX) or use real-user monitoring (RUM) which captures every actual user transaction.

Note: Switch ZCC back to your student FQDN tenant before starting this lab. Log out of the SDC tenant in ZCC (if enrolled from Lab 11) and log back in as student@<Student FQDN>.

Task 13.1 — Configure a Custom Application

1 Create the Mail_server custom app in ZDX

  1. On the Corp: Client PC, navigate to the ZIdentity landing page and click the Zscaler Digital Experience tile.
  2. In the ZDX Admin Portal, navigate to Configuration > Applications.
  3. Click + Add Custom Application.
  4. Configure:
    • Name: Mail_server
    • Status: Disable (leave disabled until probe is added)
    • Type: Custom
  5. Click Save.

Note: The custom application will remain Disabled until at least one probe is associated with it.

Task 13.2 — Create Custom Probes

2 Add the Mail_Server_Web probe

  1. Under the Mail_server application entry, click Add Probe.
  2. In the Add Probe wizard, configure:
    • Name: Mail_Server_Web
    • Status: Enable
    • Application: Mail_server
    • Probe Type: Web
    • Run Frequency: 5 minutes
    • Users / User Groups / Locations: All
    • Destination URL: https://mail.google.com
    • Number of Attempts: 2
    • Timeout: 60 seconds
    • Follow Redirect: Enable, Maximum: 5
  3. Click Next through Additional Parameters and Submit.

3 Add the Mail_Server_cloud probe

  1. Under Mail_server, click Add Probe again.
  2. Configure:
    • Name: Mail_Server_cloud
    • Status: Enabled
    • Application: Mail_server
    • Probe Type: Cloud Path
    • Follow Web Probe: Mail_Server_Web
    • Run Frequency: 5 minutes
    • Users / Groups / Locations: All
    • Protocol: ICMP
    • Packet Count: 5
    • Interval: 1000 ms
    • Timeout: 1000 ms
    • Cloud Path Host: mail.google.com
  3. Click Next and Submit.
  4. Go back to Configuration > Applications, click the edit icon for Mail_server, change Status to Enable, and click Save.
  5. Navigate to Activation and click Activate.

Result: Mail_server now shows "Enabled" with 2 active probes (Mail_Server_Web and Mail_Server_cloud). ZDX will start collecting performance data within 15–30 minutes.

MENTAL NOTE
ZDX custom app setup: ZDX Admin Portal (ZIdentity → Zscaler Digital Experience tile) → Configuration → Applications → + Add Custom Application. App is disabled until a probe is added. Web probe: name Mail_Server_Web, type Web, destination https://mail.google.com, interval 5 min. Cloud Path probe: name Mail_Server_cloud, type Cloud Path, follows Mail_Server_Web probe, protocol ICMP, host mail.google.com. Activate after creating both probes.
