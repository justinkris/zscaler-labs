LAB 11

Scenario: Explore the Zscaler Workflow Automation (ZWA) portal to triage DLP incidents. Change incident status, assign priority, notify a user for justification, test the escalation flow, and review automated workflow templates.

Time: ~35 minutes
💡 THINK FIRST · WORKFLOW AUTOMATION

ZWA centralizes DLP incident response — rather than emailing spreadsheets of violations, analysts triage incidents directly in the portal with full context: trigger data, violation content, originating user, and state history.

  • ZWA's "Validating with User" status sends an email to the violating user asking for justification. What are the risks of relying on self-justification for DLP incident resolution?
  • A workflow template maps incident attributes (rule name, user department) to automated actions. What is the advantage of attribute-based mapping over a flat "block all violations" rule?
  • ZWA integrates with SAML/SCIM to populate the Approver list based on organizational hierarchy. What happens if a user's manager is not in the directory?
Q1 — MODEL ANSWER
Self-justification creates risk because: (1) Users may provide false justifications to avoid disciplinary action. (2) Habituation — if users are asked to justify every minor violation, they learn to dismiss the process. (3) High-severity violations (intentional exfiltration) should be escalated to security/HR, not handled via user self-service. ZWA should combine self-justification for low-severity incidents with automated escalation for high-severity ones.
Q2 — MODEL ANSWER
Attribute-based mapping allows fine-grained policy: a CCN violation by a sales user in the billing department may be a false positive (legitimate business need), while the same violation by a user in R&D is high-risk. Mapping lets you route incidents to different approvers, apply different SLAs, and trigger different actions based on business context — rather than treating all violations identically.
Q3 — MODEL ANSWER
If the manager field is empty or the manager is not in the SCIM-synced directory, ZWA falls back to the configured Approver list (manual approvers added in Setup > Approvers). The Auto Escalate workflow's "Validate Manager" step returns MANAGER_NOT_FOUND and routes to the manual Approver instead of the hierarchy-based manager.

SDC Lab: This lab uses the Solutions Demo Center (SDC). For Tasks 11.1–11.5, log in to https://sdc.zslogin.net using the ZIA_ADMIN_USER_ID and Session Password from your Resources panel.

Task 11.1 — Enroll with ZCC on SDC

1 Log out of ZCC and re-enroll on SDC IT department

  1. On the Corp: Client PC, open ZCC and click Log Out (top-right corner).
  2. In the ZCC Login screen, enter the IT_department_username from the SDC Resources panel.
  3. Click Login, then at the IdP prompt enter the username again and click Next.
  4. Enter the Session_Password and click Sign in.
  5. Wait for ZCC to complete enrollment on the SDC tenant.
  6. Verify: Internet Security = ON, Network Type = Off-Trusted Network.

Note: ZCC will minimize and go through initialization steps. It may restart — wait a few minutes for all services to activate.

Task 11.2 — View Current DLP Incidents

2 Access ZWA and review the Incidents list

  1. Log in to https://sdc.zslogin.net with the ZIA_ADMIN_USER_ID and Session Password.
  2. In ZIA Admin Portal, navigate to Administration > Data Loss Prevention > Workflow Automation.
  3. The ZWA portal opens in a new tab. Click the Data Protection tab.
  4. Click Incidents to view the incident list. Review the counts: Open, Resolved, Waiting Feedback, Escalated.
  5. Click on a Transaction ID to open the Incident Details view.
  6. Review the Overview and Violation Details sections — note: DLP Admin, Priority, Action, Source DLP Type, Incident Groups.
  7. Scroll to Violation Content and click View Trigger Data to see the actual content that triggered the violation.

Task 11.3 — Modify Incident Metadata

3 Update incident status and priority

  1. On the Incident Details page, click the Actions menu (top right).
  2. Select Investigating — add a test note when prompted (e.g. "Test").
  3. From the Actions menu, select Assign Priority.
  4. Set Priority to High and click Assign.
  5. Refresh the browser and verify the Status is now "Investigating" and Priority is "High".
  6. Scroll to the State Changes section to see the full audit trail of changes.

Task 11.4 — Test User Notification and Escalation

4 Send a notification to the originating user

  1. From the Incident Details page, click Actions > Notify User.
  2. In the Notify User dialog, select Language: English.
  3. Enter a test message in the Note to user field: Please justify uploading this document for testing purposes of HandsOnLab.
  4. Click Submit.
  5. Test the escalation flow:
    • Go to Setup > Approvers > + Add More and add your name and email.
    • Return to the incident and click Actions > Escalate.
    • In the Select Approver dropdown, choose your name and click Save.
    • Check your email inbox for the escalation notification email.

Note: The notification email is sent to the originating user (not your personal email). You will receive the escalation email at the address you added as Approver.

Task 11.5 — Configure Automated Workflows

5 Explore workflow templates and mappings

  1. In the ZWA portal, click Workflows > Workflow Templates.
  2. Review the predefined templates: Auto Close, Auto Create Tickets, Auto Escalate, Auto Notify, etc.
  3. Click the View (eye) icon for the Auto Escalate template.
  4. Examine the graphical workflow: Get User Manager → Validate Manager → Escalate To Manager / Escalate To Approver.
  5. Click Workflows > Workflow Mapping.
  6. Expand the first workflow mapping to see the triggering attributes and conditions (e.g. matchingPolicies rule name CONTAINS_EXACT and userInfo.name EQUALS).
  7. Note: with read-only admin credentials, you can explore but not save changes.

Result: You understand how ZWA automates DLP incident response. Workflow templates define action sequences; Workflow Mapping determines which incidents trigger each workflow based on attribute conditions.

🧠
MENTAL NOTE
ZWA access: ZIA Admin Portal → Administration → Data Loss Prevention → Workflow Automation (opens new tab). SDC credentials required. Key tabs: Incidents (list + details), Setup (Approvers, Notification channels), Workflows (templates + mapping). Incident lifecycle: New → Investigating → Validating with User → Received Justification Response → Closed. State Changes section in incident details shows full audit trail.
